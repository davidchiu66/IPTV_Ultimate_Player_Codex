# 播放列表特性需求与实施方案

## 1. 背景与目标

当前播放器已经支持资源目录、本地媒体播放、资源收藏、频道收藏，以及播放控制板上一/下一队列切换。本轮新增“播放列表”特性，目标是为本地播放提供类似 PotPlayer 的播放列表能力：可按专辑组织本地媒体，支持持久化、自动连播、跳片头和跳片尾。

本功能只针对本地播放，不改变直播频道列表、频道收藏、资源库和浏览器播放逻辑。

## 2. 需求清单

### 2.1 播放专辑

- 初始提供一个“默认专辑”。
- 可以通过选择文件夹新建专辑。
- 新建专辑时，将该文件夹下所有支持的本地媒体加入专辑。
- 专辑需要持久化，程序重启后仍存在。
- 选择某个专辑时，加载并显示该专辑内的媒体内容。
- 每个专辑有独立设置，互不影响。

### 2.2 媒体范围

播放列表只纳入本地媒体：

- 视频
- 音频
- GIF
- 图片

自动连播第一阶段主要面向视频/音频/GIF/图片统一按队列推进；图片/GIF 的精细停留时长后续可扩展。

### 2.3 自动连播

- 从播放列表双击媒体播放后，当前播放上下文进入 `playlist_album`。
- 当前媒体自然播放结束后，自动播放当前专辑中的下一项。
- 当前专辑最后一项播放结束后，第一阶段停止，不循环。
- 播放控制板上的“上一个/下一个”在播放列表上下文中按当前专辑切换。

### 2.4 跳片头/片尾

每个专辑单独配置：

- 是否自动连播。
- 是否跳过片头。
- 片头时长，单位秒。
- 是否跳过片尾。
- 片尾时长，单位秒。

行为：

- 开始播放当前专辑媒体后，如果启用跳片头并且片头时长大于 0，自动 seek 到片头结束位置。
- 播放过程中，如果启用跳片尾并且剩余时长小于等于片尾时长，则视为当前项结束，自动播放下一项。
- 跳片尾只触发一次，避免重复切换。

### 2.5 播放列表面板

- 新增右侧滑出播放列表面板。
- 鼠标移动到界面最右边时滑入。
- 双击播放列表文件后滑出。
- 鼠标无动作 5 秒后自动滑出。
- 面板内应支持：
  - 专辑选择。
  - 新建专辑。
  - 删除专辑。
  - 专辑设置。
  - 媒体列表显示。
  - 双击播放。
  - 失效文件标注。

## 3. 数据持久化方案

新增文件：

```text
config/playlists.json
```

建议结构：

```json
{
  "version": 1,
  "active_album_id": "default",
  "albums": [
    {
      "id": "default",
      "name": "默认专辑",
      "source_dir": "",
      "recursive": false,
      "items": [],
      "settings": {
        "auto_play_next": true,
        "skip_intro": false,
        "intro_seconds": 0,
        "skip_outro": false,
        "outro_seconds": 0
      },
      "created_at": "",
      "updated_at": ""
    }
  ]
}
```

新增管理器：

```text
backend/playlist_album_manager.py
```

职责：

- 加载和保存播放专辑。
- 创建默认专辑。
- 从文件夹创建专辑。
- 扫描本地媒体。
- 删除专辑、更新专辑设置。
- 校验媒体文件存在性。
- 获取当前专辑上一项/下一项。

## 4. UI 方案

新增：

```text
ui/playlist_overlay.py
ui/playlist_album_settings_dialog.py
```

`PlaylistOverlay`：

- 继承现有 `BaseOverlay`，从右侧滑出。
- 顶部为专辑选择区域和新增按钮。
- 中部为当前专辑媒体列表。
- 底部为添加、删除、设置、刷新等操作按钮。
- 鼠标活动重置 5 秒自动隐藏计时器。

`PlaylistAlbumSettingsDialog`：

- 专辑名称。
- 来源文件夹。
- 是否包含子文件夹。
- 是否自动连播。
- 是否跳过片头。
- 片头时长。
- 是否跳过片尾。
- 片尾时长。

## 5. 播放集成方案

### 5.1 右边缘触发

在 `PlayerPanel` 增加：

```python
right_edge_entered = Signal()
```

并新增右侧 20px 触发区，鼠标进入后通知 `MainWindow` 显示播放列表面板。

### 5.2 播放入口

在 `MainWindow` 增加播放列表媒体入口：

```python
_on_playlist_item_selected(album_id, item_id, path)
```

入口会调用现有本地媒体播放逻辑，并设置播放上下文：

```text
kind = playlist_album
current_key = item_id
source = album_id
```

### 5.3 上一/下一

扩展现有 `_play_adjacent_item()`：

- 当上下文为 `playlist_album` 时，从当前专辑里找上一项/下一项。
- 控制板按钮不再落到资源目录或资源收藏队列。

### 5.4 自动连播

在 `MpvVideoWidget` 增加本地媒体结束信号：

```python
local_media_finished = Signal(dict)
```

检测方式：

- 当前是本地媒体。
- `eof-reached` 为 True。
- 或 `idle-active` 为 True。
- 或 `time-pos >= duration - 0.35`。

在 `MainWindow` 连接该信号，如果当前上下文是 `playlist_album` 且专辑启用自动连播，则播放下一项。

### 5.5 跳片头/片尾

- `playback_started` 后，如果当前上下文是 `playlist_album` 且专辑启用跳片头，则 seek 到片头时长位置。
- 播放进度更新时，如果当前上下文是 `playlist_album` 且专辑启用跳片尾，剩余时长小于等于片尾时长，则自动播放下一项。

## 6. 验收办法

1. 首次启动后，播放列表中存在“默认专辑”。
2. 鼠标移动到界面最右侧，播放列表面板滑入。
3. 通过文件夹新建专辑后，媒体文件出现在专辑列表内。
4. 重启后，新建专辑仍存在。
5. 双击播放列表中的本地视频，视频开始播放，播放列表面板滑出。
6. 鼠标无动作 5 秒，播放列表面板自动滑出。
7. 播放列表上下文中，控制板“下一个”播放当前专辑下一项。
8. 播放列表上下文中，控制板“上一个”播放当前专辑上一项。
9. 视频自然播放结束后，自动播放下一项。
10. 专辑设置跳过片头 60 秒后，开始播放时自动跳到 60 秒附近。
11. 专辑设置跳过片尾 90 秒后，剩余 90 秒左右自动切到下一项。
12. 专辑 A 和专辑 B 的跳片头/片尾设置互不影响。
13. 删除专辑不会删除原始媒体文件。
14. 文件不存在时，列表项标注失效，播放时给出友好提示。
15. 直播播放时，播放列表不影响频道列表、资源库和直播播放。

## 7. 后续增强

- 支持拖拽排序。
- 支持随机播放、单项循环、列表循环。
- 支持图片/GIF 独立停留时长。
- 支持导入/导出播放列表。
- 支持按当前筛选结果连播。
- 支持播放历史和断点续播。
