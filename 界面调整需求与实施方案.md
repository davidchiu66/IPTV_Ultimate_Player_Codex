# 界面调整需求与实施方案

## 背景

当前界面存在多个入口职责重叠：

- 顶部工具栏按钮过多，包含全局操作、播放操作、设置操作和节目单操作。
- 播放状态顶部条已有频道详情、轨道、清晰度等播放中操作，但部分常用操作仍分散在顶部工具栏。
- 左侧滑出当前是频道文件列表，而用户预期播放时左侧更适合快速切换频道分组和频道列表。

本轮目标是调整入口职责，不丢失已有功能。

## 总体原则

1. 当前已有功能不能丢失，只调整入口和布局。
2. 顶部工具栏只保留全局入口。
3. 播放状态顶部条承载播放中常用功能。
4. 左侧边缘滑出承载当前已加载文件内的频道分组和频道列表。
5. 设置改为右侧滑出面板，便于后续扩展。
6. 本文档经确认后再进入编码实施。

## 需求清单

### 1. 顶部工具栏调整

顶部工具栏只保留以下按钮：

- `打开频道资源`
- `打开在线 m3u`
- `设置`

原工具栏上的其他按钮全部移除入口，但对应功能保留并迁移到合适位置。

`打开频道资源` 按钮放置在工具栏最左侧，样式与播放状态顶部条左侧菜单按钮保持一致：

- 使用同类三横线/列表菜单图标。
- 优先使用用户上传的图标资源；如不便直接引用，则使用 Qt 绘制或文本图标兜底。
- 视觉上作为工具栏的主入口图标按钮，而不是普通长文字按钮。
- 保留 tooltip 或可访问文本为 `打开频道资源`。

`设置` 按钮放置在工具栏最右侧：

- 使用齿轮图标 `⚙️` 显示。
- 视觉上作为工具栏的设置入口图标按钮，而不是普通长文字按钮。
- 保留 tooltip 或可访问文本为 `设置`。

### 2. 打开频道资源

`打开频道资源` 按钮用于完成当前频道文件列表加载功能：

- 刷新当前 `Channels` 目录或当前频道资源目录。
- 滑出频道文件列表清单。
- 继续支持打开目录、刷新、删除、双击加载频道文件。

### 3. 播放顶部条菜单按钮调整

播放状态顶部条左侧菜单按钮原来打开频道列表，本轮调整为：

- 打开当前加载文件列表清单。
- 也就是显示现有 `NavigationOverlay` 的频道文件列表面板。

### 4. 左侧滑出行为调整

鼠标移动到最左侧滑出的内容调整为：

- 频道分组。
- 频道列表。
- 搜索当前已加载频道。
- 点击频道后播放。

也就是显示现有 `ChannelListOverlay`。

### 5. 设置面板

点击顶部工具栏 `设置` 按钮后，从界面右侧滑出应用设置界面。

首批设置项：

- 系统代理：只读显示。
- 用户配置代理：输入框，可留空。
- 当前生效代理：只读显示。
- 浏览器探测超时时长：可配置。
- 浏览器播放端口：可配置。

操作按钮：

- `保存`：写入配置文件并关闭设置面板。
- `取消`：放弃本次修改并关闭设置面板。

后续如有需要，可继续扩展更多设置项。

### 6. 播放顶部条新增按钮

播放状态顶部条新增：

- `停止播放`
- `全屏 F1`
- `加载节目单`
- `下载节目单`

对应复用现有功能：

- `停止播放` -> `stop_playback`
- `全屏 F1` -> `_toggle_fullscreen`
- `加载节目单` -> `load_local_epg`
- `下载节目单` -> `download_epg_dialog`

快捷键要求：

- 新增 `F1` 切换全屏。
- 保留现有 `F11` 全屏切换，避免旧习惯失效。

### 7. 顶部条保留频道详情

播放状态顶部条必须保留现有 `频道详情` 按钮。

现有链路保持：

- `PlayerTopBar.detail_clicked`
- `MainWindow._show_detail`
- `DetailOverlay`

本轮不移除、不替换、不降级频道详情功能。

### 8. 停止播放后的状态

点击 `停止播放` 后，应回到最初状态：

- 停止 libmpv 播放。
- 播放画面消失。
- 加载动画消失。
- 顶部播放条消失。
- 底部播放控制条消失。
- 回到初始空闲/黑屏占位状态。
- 不保留当前播放态控件浮层。

## 当前代码结构分析

### 顶部工具栏

相关文件：

- `ui/toolbar_overlay.py`
- `ui/main_window.py`

当前 `MainWindow` 在初始化时向 `ToolbarOverlay` 添加了多个按钮，包括保存、节目单、浏览器播放、代理、停止、全屏、探测超时等。需要收敛为三个全局入口。

### 频道文件列表

相关文件：

- `ui/navigation_overlay.py`
- `ui/main_window.py`

当前 `NavigationOverlay` 已经承载频道文件列表清单，可复用为 `打开频道资源` 和顶部条菜单按钮的目标面板。

### 频道分组和频道列表

相关文件：

- `ui/channel_list_overlay.py`
- `ui/main_window.py`

当前 `ChannelListOverlay` 已经是“分组 + 频道列表”结构，可复用为左侧边缘滑出面板。

### 播放顶部条

相关文件：

- `ui/player_controls.py`
- `ui/player_panel.py`
- `ui/main_window.py`

当前 `PlayerTopBar` 已有：

- 菜单按钮。
- 频道标题和分组。
- 清晰度。
- 视频轨道。
- 音频轨道。
- 字幕轨道。
- 频道详情。

需要新增播放操作按钮，同时保留频道详情。

### 设置

相关文件：

- `utils/proxy_settings.py`
- `config/app_settings.json`
- 新增建议：`ui/settings_overlay.py`

当前代理和探测超时通过弹窗配置，浏览器端口只在运行期变量 `server_port` 中维护。本轮改为统一右侧滑出设置面板，并将浏览器端口纳入持久配置。

## 实施方案

### 阶段 1：文档与遗留问题

1. 新增 `遗留问题清单.md`。
2. 记录 iptv345.com m3u8 已嗅探但 libmpv 404 的问题。
3. 新增本方案文档，作为本轮 UI 调整实施依据。

### 阶段 2：工具栏入口收敛

1. 修改 `MainWindow` 初始化按钮注册。
2. 顶部工具栏仅添加：
   - 打开在线 m3u
   - 打开频道资源
   - 设置
3. `打开频道资源` 放置在最左侧，并使用与播放顶部条左侧菜单按钮一致的图标按钮样式。
4. `设置` 放置在最右侧，并使用齿轮图标按钮样式。
5. 保留原功能函数，不删除业务代码。
6. `打开频道资源` 连接到刷新并显示文件列表面板。

### 阶段 3：滑出面板入口重映射

1. 播放顶部条菜单按钮连接 `_show_navigation`。
2. 左侧边缘触发连接 `_show_channel_list`。
3. 确保两个面板互斥显示，不互相遮挡。
4. 更新按钮 tooltip 和显示文本。

### 阶段 4：播放顶部条功能扩展

1. 在 `PlayerTopBar` 增加信号：
   - `stop_clicked`
   - `fullscreen_clicked`
   - `load_epg_clicked`
   - `download_epg_clicked`
2. 增加按钮：
   - 停止播放
   - 全屏 F1
   - 加载节目单
   - 下载节目单
3. 保留频道详情按钮和原有轨道/清晰度控件。
4. 在 `PlayerPanel` 和 `MainWindow` 接好对应信号。

### 阶段 5：停止播放恢复初始状态

1. 修改 `PlayerPanel.stop_playback()`。
2. 停止 mpv 后切回 `placeholder_widget`。
3. 隐藏加载层。
4. 隐藏顶部条和底部条。
5. 清理播放中状态文字。
6. 保持频道列表数据不清空，方便再次播放。

### 阶段 6：右侧设置面板

1. 新增 `ui/settings_overlay.py`。
2. 继承 `BaseOverlay(side="right")`。
3. 增加代理、探测超时、浏览器端口字段。
4. 保存时写入 `config/app_settings.json`。
5. 取消时关闭面板不写入。
6. `MainWindow` 启动时读取浏览器端口配置。

### 阶段 7：配置工具函数补充

在 `utils/proxy_settings.py` 增加或整理：

- 获取/保存用户代理。
- 获取/保存探测超时。
- 获取/保存浏览器端口。
- 获取系统代理和当前生效代理。

## 验收方法

1. 启动程序后，顶部工具栏只有：
   - 打开在线 m3u
   - 打开频道资源
   - 设置
2. `打开频道资源` 位于工具栏最左侧，样式与播放顶部条左侧菜单按钮一致。
3. `设置` 位于工具栏最右侧，显示为齿轮图标 `⚙️`。
4. 点击 `打开频道资源` 后，显示频道文件列表清单。
5. 播放状态顶部条左侧菜单按钮显示频道文件列表清单。
6. 鼠标移动到最左边，显示频道分组和频道列表。
7. 顶部条 `频道详情` 按钮仍能打开频道详情。
8. 顶部条 `停止播放` 后，播放画面、顶部条、底部条都消失，回到初始状态。
9. 顶部条 `全屏 F1` 可切换全屏，`F11` 仍可用。
10. 顶部条 `加载节目单` 和 `下载节目单` 功能与原来一致。
11. 点击顶部工具栏 `设置`，右侧设置面板滑出。
12. 设置面板保存后代理、探测超时、浏览器端口生效并写入配置。
13. 设置面板取消后不写入配置。
14. `python -m py_compile` 校验通过。

## 风险点

- 顶部条按钮增多后可能空间不足，需要控制按钮宽度或使用紧凑文本。
- 设置面板保存浏览器端口后，如果 HTTP 服务已启动，需要提示下次启动生效或重启本地服务。
- 停止播放恢复初始状态时，要避免误清空当前频道文件和频道列表。
- 入口重映射后，需要确认触发区和覆盖层 z-order 不冲突。
