# 左边缘面板触发策略调整需求与实施方案

## 1. 背景

当前播放器在沉浸播放状态下，鼠标移动到最左边时固定触发 `频道列表面板` 滑入。这个策略适合直播频道播放场景，因为用户通常希望快速切换上一个/下一个频道或在频道列表中选择其他频道。

但在本地资源播放场景下，包括本地视频、音频、GIF、图片等，左边缘继续滑出频道列表并不符合用户预期。此时用户更可能需要回到 `资源面板`，继续选择其他本地资源、频道资源文件或收藏资源。

因此需要调整左边缘触发策略：

- 只有在直播播放状态下，鼠标移动到最左边才滑出 `频道列表面板`。
- 其他情况下，鼠标移动到最左边都滑出 `资源面板`。

## 2. 当前实现分析

当前关键接线位于 `ui/main_window.py`：

```text
self.player_panel.left_edge_entered.connect(self._show_channel_list)
```

也就是说，左边缘触发无条件调用 `_show_channel_list()`。

现有可复用能力：

- `MainWindow._show_channel_list()`：显示频道列表面板。
- `MainWindow._show_navigation()`：显示资源面板。
- `utils.media_types.is_local_media_channel(channel)`：判断当前频道是否为本地资源。
- 本地资源播放时会构造频道快照：

```text
ManifestType = "local"
_IsLocalMedia = True
```

需要注意：

- `MainWindow.current_channel` 在停止播放后可能仍保留上一条频道。
- 因此不能只根据 `current_channel` 判断是否处于直播播放状态。
- 需要同时参考播放器当前是否有活动播放实例。真实 libmpv 播放实例位于 `self.player_panel.mpv_widget.player`，不能误用外层 `self.player_panel.player`。

## 3. 需求定义

### 3.1 左边缘触发规则

鼠标移动到播放器最左边时：

| 当前状态 | 目标面板 |
| --- | --- |
| 正在播放直播频道 | 频道列表面板 |
| 正在播放本地视频 | 资源面板 |
| 正在播放本地音频 | 资源面板 |
| 正在播放 GIF | 资源面板 |
| 正在播放图片 | 资源面板 |
| 当前没有播放内容 | 资源面板 |
| 已停止播放 | 资源面板 |
| 播放失败后处于空闲状态 | 资源面板 |

### 3.2 直播播放状态定义

建议定义为：

```text
真实 mpv widget 播放器实例存在
并且 current_channel 存在
并且 current_channel 不是本地资源
```

等价伪代码：

```python
def _is_live_playback_context(self):
    mpv_widget = getattr(self.player_panel, "mpv_widget", None)
    if mpv_widget is None or getattr(mpv_widget, "player", None) is None:
        return False
    if getattr(mpv_widget, "_failure_sent", False):
        return False
    channel = self.current_channel or getattr(mpv_widget, "current_channel", None) or {}
    if not channel:
        return False
    if is_local_media_channel(channel):
        return False
    return True
```

说明：

- 这里的“直播”主要指非本地资源频道，包括 HLS、DASH、RTMP、RTSP、FLV、页面嗅探后得到的直播源等。
- 如果未来支持远程点播/VOD，可再增加 `is_live_channel()` 判断，本轮先沿用现有 IPTV 场景。

## 4. 实施方案

### 4.1 新增左边缘分流函数

在 `MainWindow` 中新增方法：

```python
def _show_left_edge_panel(self):
    if self._is_live_playback_context():
        self._show_channel_list()
    else:
        self._show_navigation()
```

### 4.2 新增直播上下文判断函数

在 `MainWindow` 中新增方法：

```python
def _is_live_playback_context(self):
    mpv_widget = getattr(self.player_panel, "mpv_widget", None)
    if mpv_widget is None or getattr(mpv_widget, "player", None) is None:
        return False
    if getattr(mpv_widget, "_failure_sent", False):
        return False
    channel = self.current_channel or getattr(mpv_widget, "current_channel", None) or {}
    if not channel:
        return False
    if is_local_media_channel(channel):
        return False
    return True
```

### 4.3 修改信号连接

将当前左边缘接线：

```python
self.player_panel.left_edge_entered.connect(self._show_channel_list)
```

改为：

```python
self.player_panel.left_edge_entered.connect(self._show_left_edge_panel)
```

### 4.4 保持其他入口不变

以下功能不改变：

- 播放控制条菜单按钮仍打开资源面板。
- 顶部工具栏资源按钮仍打开资源面板。
- 频道列表内部点击频道仍播放频道。
- 资源面板双击资源仍按现有逻辑加载或播放。
- 直播播放时左边缘仍保持快速切换频道体验。

## 5. 风险与注意事项

- 停止播放后如果只看 `current_channel`，可能误判为直播状态，因此必须同时判断 `mpv_widget.player` 是否存在。
- 本地资源播放时可能仍保留频道列表数据，但左边缘不应显示频道列表。
- 如果播放失败进入空闲状态，左边缘应回到资源面板，方便用户换资源。
- `_show_navigation()` 与 `_show_channel_list()` 已经处理互斥隐藏，分流函数应复用它们，不重复写隐藏逻辑。
- 后续如果增加远程点播类型，需要再细分“直播频道”和“远程媒体资源”。

## 6. 验收方法

### 用例 1：直播播放时左边缘显示频道列表

1. 加载一个频道资源文件。
2. 播放一个直播频道。
3. 鼠标移动到播放器最左边。

预期结果：

- 滑出频道列表面板。
- 不滑出资源面板。
- 可以继续在频道列表中切换频道。

### 用例 2：本地视频播放时左边缘显示资源面板

1. 打开资源目录。
2. 双击播放一个本地视频。
3. 鼠标移动到播放器最左边。

预期结果：

- 滑出资源面板。
- 不滑出频道列表面板。
- 可以继续选择其他资源文件。

### 用例 3：本地音频/GIF/图片播放时左边缘显示资源面板

1. 分别播放本地音频、GIF、图片资源。
2. 每次播放后鼠标移动到播放器最左边。

预期结果：

- 都滑出资源面板。
- 不滑出频道列表面板。

### 用例 4：停止播放后左边缘显示资源面板

1. 播放一个直播频道。
2. 点击停止播放。
3. 鼠标移动到播放器最左边。

预期结果：

- 滑出资源面板。
- 不因为 `current_channel` 仍保留上一条直播频道而误滑频道列表。

### 用例 5：无播放内容时左边缘显示资源面板

1. 启动程序，不播放任何内容。
2. 鼠标移动到播放器最左边。

预期结果：

- 滑出资源面板。
- 不滑出空频道列表。

### 用例 6：播放失败后左边缘显示资源面板

1. 播放一个会失败或不可用的资源。
2. 等待失败弹窗或失败状态出现。
3. 鼠标移动到播放器最左边。

预期结果：

- 滑出资源面板。
- 用户可以快速选择其他资源重试。

## 7. 编译与基础验证

修改后至少执行：

```text
python -m py_compile ui/main_window.py ui/player_panel.py utils/media_types.py
```

并建议执行一次短时启动构造测试，确认启动不受影响。

## 8. 结论

本需求属于交互分流调整，不需要改动播放核心、资源解析或收藏持久化。推荐实现方式是在 `MainWindow` 增加一个左边缘分流函数，把原本固定显示频道列表的入口改为根据播放上下文选择频道列表或资源面板。
