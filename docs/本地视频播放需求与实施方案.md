# 本地视频播放需求与实施方案

## 需求背景

当前资源库面板主要面向频道配置文件，目录扫描只识别 cfg、json、m3u、m3u8、txt 等频道资源文件。用户希望把“打开频道目录”升级为“打开资源目录”，同一个目录中既可以加载频道文件，也可以加载本地视频文件。对于本地视频文件，双击后直接使用内置 libmpv 播放。

## 需求清单

1. 将资源库面板中的“打开频道目录”修改为“打开资源目录”。
2. 资源目录可以同时扫描频道资源文件和本地媒体文件。
3. 频道资源文件保持原有行为：双击后加载频道列表。
4. 本地视频文件采用新行为：双击后直接调用 libmpv 播放。
5. 本地视频播放不经过 HTTP 检测、网页嗅探、直播源解析，也不使用浏览器兜底。
6. 本地视频播放失败时，提示本地媒体播放失败、文件不存在或格式不支持等原因。
7. 现有频道播放、资源库面板、顶部条、频道列表等功能不能丢失。

## 支持范围

### 频道资源文件

- cfg
- json
- m3u
- m3u8
- txt

### 本地媒体文件

第一阶段以 libmpv 常见视频格式为主：

- mp4
- mkv
- avi
- mov
- wmv
- flv
- webm
- ts
- m2ts
- mts
- mpg
- mpeg
- m4v
- 3gp
- ogv

可预留音频格式支持，但本阶段重点是本地视频播放。

## 实施方案

1. 新增统一媒体类型工具模块，集中维护频道资源扩展名、本地媒体扩展名、资源类型判断等方法。
2. 调整资源库面板文案：
   - “打开频道目录”改为“打开资源目录”
   - “频道文件”改为“资源文件”
   - “文件数”改为“资源数”
3. 调整目录扫描逻辑：
   - 同时扫描频道资源文件和本地媒体文件。
   - 列表展示时增加类型前缀，例如 `[频道] US.cfg`、`[视频] sample.mkv`。
4. 调整双击资源文件逻辑：
   - 如果是频道资源文件，沿用原有 `load_config()` 流程。
   - 如果是本地媒体文件，构造临时频道对象，字段包括 `Name`、`Manifest`、`ManifestType=local`、`_IsLocalMedia=True`，并直接播放。
5. 调整播放器逻辑：
   - 识别本地媒体。
   - 本地媒体跳过 `resolve_channel()`。
   - 本地媒体的流类型标记为 `local`。
   - 加载提示显示为“正在打开本地媒体...”。
6. 调整失败处理：
   - 本地媒体 `loadfile` 失败时，失败动作标记为 `local-failed`。
   - 主窗口收到本地媒体失败时，不展示浏览器兜底选择，仅提示本地媒体无法播放。

## 验收方法

1. 打开资源目录后，资源库面板能同时显示频道文件和本地视频文件。
2. 双击 cfg、m3u、txt 等频道文件，仍能加载频道列表。
3. 双击 mp4、mkv、flv、ts 等本地视频文件，能直接进入播放状态并由 libmpv 播放。
4. 本地视频路径包含空格或中文时，仍能正常播放。
5. 本地视频播放失败时，不弹出浏览器播放选项。
6. 原有 HLS、MPD、MPD/CENC 频道播放链路不受影响。

## 风险与后续

1. 少数容器或编码虽然扩展名支持，但本机 libmpv/ffmpeg 仍可能因解码能力不足播放失败。
2. 本阶段不做本地字幕自动挂载，后续可支持同名 srt、ass、ssa 字幕自动加载。
3. 后续可继续扩展本地音频播放、播放历史、最近打开资源目录等能力。
