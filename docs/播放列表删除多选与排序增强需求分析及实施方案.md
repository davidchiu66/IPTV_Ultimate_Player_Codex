# 播放列表删除、多选与排序增强需求分析及实施方案

## 背景

当前右侧播放列表面板已经支持播放列表专辑切换、新建、设置、刷新、删除专辑、双击播放条目和播放记忆点。但现有交互中，“删除”按钮实际用于删除播放列表专辑，和用户对“移除列表中选定文件”的直觉不一致；同时列表条目只能单选播放，缺少手动添加文件、多选移除和手动排序能力。

本次目标是在不破坏现有播放列表数据结构和播放逻辑的前提下，增强播放列表管理能力：

1. 在播放列表名称下拉框右侧，保留 `+` 新增按钮，并新增 `-` 删除播放列表按钮。
2. 在“移除”按钮前新增“添加”按钮，用于向当前播放列表添加指定目录下的一个或多个媒体文件。
3. 将原底部“删除”按钮改名为“移除”，用于从当前播放列表中移除选定条目。
4. 支持按住 `Ctrl` 多选条目，被选中条目应显示高亮背景。
5. 新增“置顶、上移、下移、置底”排序按钮，用于调整选定条目顺序。
6. 添加、移除和排序结果必须保存，下次打开或下次启动按最新列表加载。
7. 新增按钮风格遵循当前播放列表面板统一玻璃拟态按钮样式。

## 当前代码分析

相关文件：

1. `ui/playlist_overlay.py`
   - 负责右侧播放列表 UI。
   - 已有 `album_combo`、`new_button`、`settings_button`、`refresh_button`、`delete_button`、`memory_button`。
   - 当前 `delete_button` 绑定 `_emit_delete_album()`，用于删除播放列表专辑。
   - `item_list` 是 `QListWidget`，当前未显式开启多选。
   - 已有 `QListWidget::item:selected` 样式，开启多选后可复用现有高亮背景。

2. `backend/playlist_album_manager.py`
   - 播放列表持久化到 `playlists.json`。
   - 每个专辑的 `items` 数组天然保存条目顺序。
   - `scan_directory()` 当前按文件名排序，用于新建和刷新。
   - `update_album(..., {"rescan": True})` 当前会用扫描结果整体替换 `items`，会丢失用户手动排序。
   - 已有 `delete_album()`，但没有添加条目、移除条目或移动条目的方法。

3. `ui/main_window.py`
   - 连接播放列表面板信号。
   - `_delete_playlist_album()` 已实现删除专辑确认。
   - `_refresh_playlist_album()` 当前直接触发重扫。
   - 相邻播放 `_play_adjacent_playlist_album()` 依赖 `PlaylistAlbumManager.adjacent_item()`，该方法按 `items` 当前顺序计算上一项/下一项，因此只要保存顺序即可影响播放顺序。

## 需求边界

### 移除选定文件的语义

建议本次“移除列表中选定文件”解释为：从当前播放列表中移除选定条目，不删除磁盘上的真实媒体文件。

理由：

1. 新增 `-` 按钮删除播放列表本身也不删除原始媒体文件，列表条目操作则明确叫“移除”，保持语义清晰。
2. 物理删除媒体文件风险更高，应单独确认和设计。
3. 播放列表是专辑关系管理，条目删除应优先理解为“从列表移除”。

### 默认播放列表

沿用现有规则：默认播放列表不能删除。因此新增 `-` 删除播放列表按钮在默认播放列表下点击时提示“默认播放列表不能删除”，或直接置灰。

### 多选范围

使用 `QListWidget.ExtendedSelection`：

1. `Ctrl + 点击` 可多选/取消多选。
2. `Shift + 点击` 可连续选择。
3. 单击仍保持普通选择。
4. 双击任意条目仍播放该条目。

### 添加文件的语义

“添加”按钮用于向当前播放列表追加一个或多个本地媒体文件：

1. 通过文件选择对话框选择文件，支持一次选择多个文件。
2. 只添加 `utils.media_types.is_local_media()` 认可的本地媒体文件。
3. 添加动作只修改播放列表关系，不复制、不移动、不改名原始文件。
4. 默认追加到当前播放列表末尾，保持用户已有手动排序不变。
5. 如果所选文件已经在当前播放列表中，不重复添加。
6. 如果所选文件此前被“移除”并记录在 `excluded_paths` 中，本次添加应从 `excluded_paths` 移除，并重新追加到列表末尾。

## 实施方案

### 1. 调整播放列表面板按钮布局

在 `PlaylistOverlay` 中调整顶部播放列表选择行：

1. `album_combo` 后保留 `+` 新建按钮。
2. 在 `+` 后新增 `-` 删除播放列表按钮。
3. `-` 按钮使用 `dangerButton` 或轻量危险按钮样式，宽度与 `+` 一致。
4. 将删除播放列表的信号从原 `delete_button` 迁移到新增 `album_delete_button`。

调整底部功能区：

1. 在“移除”按钮前新增“添加”按钮，用于打开本地媒体文件多选对话框。
2. 原“删除”按钮改名为“移除”，用于从当前列表移除选中条目。
3. 新增一行排序按钮：“置顶 / 上移 / 下移 / 置底”。
4. 排序按钮和“添加”按钮使用当前 `QPushButton` 通用样式，不新增突兀视觉。
5. “移除”可以继续使用 `dangerButton`，但按钮文案需避免与删除播放列表混淆。

### 2. 开启条目多选和状态管理

在 `PlaylistOverlay` 中：

1. `self.item_list.setSelectionMode(QListWidget.ExtendedSelection)`。
2. 复用现有 `QListWidget::item:selected` 样式，确保多选条目均高亮。
3. 新增辅助方法 `_selected_item_ids()`，从选中项中读取 `Qt.UserRole` 的 `id`。
4. 在刷新列表后可按传入的 item id 集合恢复选中状态，方便删除/移动后保持用户定位。

### 3. 增加 UI 信号

在 `PlaylistOverlay` 中新增信号：

1. `add_items_requested = Signal(str)`，参数为 `album_id`。
2. `remove_items_requested = Signal(str, list)`，参数为 `album_id, item_ids`。
3. `move_items_requested = Signal(str, list, str)`，参数为 `album_id, item_ids, action`。

`action` 建议取值：

1. `top`
2. `up`
3. `down`
4. `bottom`

### 4. 增加播放列表数据操作方法

在 `PlaylistAlbumManager` 中新增方法：

1. `add_items(album_id, paths) -> int`
   - 规范化传入文件路径。
   - 过滤不存在文件、目录和不支持的媒体类型。
   - 对当前列表中已存在的路径去重。
   - 对此前记录在 `excluded_paths` 的路径解除排除。
   - 将新增条目追加到 `items` 末尾。
   - 更新 `updated_at` 并保存 `playlists.json`。
   - 返回实际新增数量。

2. `remove_items(album_id, item_ids) -> int`
   - 从目标专辑 `items` 中移除匹配 id 的条目。
   - 更新 `updated_at`。
   - 如果播放记忆点指向被删条目，清空该专辑 `playback_memory`。
   - 保存 `playlists.json`。
   - 返回实际删除数量。

3. `move_items(album_id, item_ids, action) -> bool`
   - 在目标专辑 `items` 内移动选中条目。
   - 多选移动时保持选中条目之间的相对顺序。
   - `top`：选中条目整体移动到列表顶部。
   - `bottom`：选中条目整体移动到列表底部。
   - `up`：选中条目整体上移一格；如果最靠前的选中条目已经在顶部，则不移动。
   - `down`：选中条目整体下移一格；如果最靠后的选中条目已经在底部，则不移动。
   - 更新 `updated_at` 并保存。

### 5. 刷新/重扫保留手动排序

调整 `update_album(..., {"rescan": True})` 的行为：

1. 扫描目录得到 `scanned_items`。
2. 按路径 key 建立现有条目映射。
3. 对仍然存在的旧条目，按当前 `items` 顺序保留。
4. 对新增文件，按扫描排序追加到末尾。
5. 对旧列表中仍存在且仍受支持的媒体文件继续保留，即使它不是当前目录扫描新命中的文件，以便手动添加的外部文件能持久保留。
6. 对已不存在或不再支持的文件，从列表中移除。
7. 对用户手动移除但磁盘仍存在的文件，记录到 `excluded_paths`，刷新时继续跳过，避免被重新扫回列表。

建议本次采用“保留现有顺序 + 新文件追加末尾 + 已不存在文件移除 + 手动移除文件排除”的策略，与当前刷新“重新扫描有效文件”的行为接近，同时不破坏手动排序和手动移除结果。

### 6. 主窗口连接与反馈

在 `MainWindow` 中：

1. 连接 `playlist_overlay.add_items_requested` 到 `_add_playlist_items(album_id)`。
2. 连接 `playlist_overlay.remove_items_requested` 到 `_remove_playlist_items(album_id, item_ids)`。
3. 连接 `playlist_overlay.move_items_requested` 到 `_move_playlist_items(album_id, item_ids, action)`。
4. 添加文件时打开 `QFileDialog.getOpenFileNames()`，过滤为项目支持的本地媒体文件。
5. 移除条目前弹窗确认，提示“仅从播放列表移除，不删除原始媒体文件”。
6. 添加、移除或移动成功后刷新播放列表面板，并在状态栏提示结果。
7. 如果正在播放的条目被移除，不强制停止当前播放；但下一项/上一项会按最新列表顺序工作。

## 数据兼容性

不需要升级 `playlists.json` schema version，但会增加一个向后兼容的可选字段：

1. 排序结果直接体现在现有 `albums[].items` 数组顺序中。
2. 添加条目是向现有数组追加规范化 item。
3. 移除条目是从现有数组移除。
4. `albums[].excluded_paths` 记录从列表移除但仍存在于磁盘的文件路径，避免刷新时重新扫回；手动添加同一路径时解除排除。
5. 旧配置文件没有 `excluded_paths` 时按空列表处理，仍按原数组顺序显示。

## 风险与处理

1. 风险：刷新播放列表后用户排序丢失。
   - 处理：重扫逻辑保留旧顺序，仅追加新增文件。

2. 风险：多选移动后选择状态丢失。
   - 处理：移动后刷新面板并恢复被移动条目的选中状态。

3. 风险：添加已存在文件导致列表重复。
   - 处理：按规范化路径 key 去重，已存在文件不重复插入。

4. 风险：用户移除过的文件在刷新后被重新扫回。
   - 处理：移除时写入 `excluded_paths`；手动添加时再解除排除。

5. 风险：移除选中条目被误解为删除磁盘文件。
   - 处理：确认弹窗明确写明“不删除原始媒体文件”。

6. 风险：当前正在播放条目被删除后自动连播找不到当前位置。
   - 处理：当前播放不中断；后续上一项/下一项按最新列表查找，找不到当前 id 时沿用现有兜底逻辑。

7. 风险：默认播放列表删除入口混淆。
   - 处理：默认播放列表下禁用 `-` 或点击提示不能删除。

## 验证方法

### 场景 1：删除播放列表

1. 新建一个非默认播放列表。
2. 在播放列表名称右侧点击 `-`。
3. 确认弹窗出现，文案说明不删除原始媒体文件。
4. 确认后播放列表从下拉框消失。
5. 默认播放列表下点击 `-` 时，应提示不能删除或按钮置灰。

### 场景 2：单选移除条目

1. 打开一个包含多个媒体条目的播放列表。
2. 单击选中一个条目，确认背景高亮。
3. 点击“移除”。
4. 确认后该条目从列表中消失。
5. 重启程序或重新打开播放列表，确认该条目仍不再出现。
6. 确认磁盘上的原始媒体文件仍存在。

### 场景 3：添加一个或多个文件

1. 打开一个播放列表。
2. 点击“添加”。
3. 在文件选择对话框中选择一个或多个本地媒体文件。
4. 确认后新文件追加到播放列表末尾。
5. 重启程序或重新打开播放列表，确认新增条目仍存在。
6. 再次添加同一文件，确认不会重复出现。
7. 对曾经“移除”的文件执行添加，确认该文件重新出现在列表末尾，刷新后不会消失。

### 场景 4：Ctrl 多选移除条目

1. 按住 `Ctrl` 选择多个不连续条目。
2. 多个选中条目均应高亮。
3. 点击“移除”。
4. 确认后所有选中条目从列表移除。
5. 重启后顺序和删除状态保持。

### 场景 5：排序按钮

1. 选中一个条目，点击“置顶”，该条目移动到第一位。
2. 选中一个条目，点击“置底”，该条目移动到最后一位。
3. 选中一个条目，点击“上移 / 下移”，该条目每次移动一位。
4. 按住 `Ctrl` 多选多个条目，点击“置顶 / 置底”，多个条目的相对顺序保持不变。
5. 多选相邻条目点击“上移 / 下移”，整体移动一位，相对顺序保持不变。

### 场景 6：排序持久化

1. 手动调整播放列表顺序。
2. 关闭并重新打开播放列表面板，顺序保持。
3. 重启程序，顺序保持。
4. 播放某条目后点击下一项，按最新排序进入下一条。

### 场景 7：刷新播放列表保序

1. 手动调整顺序。
2. 点击“刷新”。
3. 已存在条目保持手动顺序。
4. 新增媒体文件追加到末尾。
5. 已删除的磁盘文件不再出现在列表中。

## 建议实施顺序

1. 修改 `PlaylistAlbumManager`，增加添加条目、移除条目、移动条目和刷新保序能力。
2. 修改 `PlaylistOverlay`，调整按钮布局、开启多选、增加添加/移除/移动信号和排序按钮。
3. 修改 `MainWindow`，连接新信号并实现文件选择、确认、调用、刷新、状态栏反馈。
4. 运行 Python 编译检查。
5. 使用临时 `PlaylistAlbumManager` 存储文件做数据层单元式验证。
6. 进行界面手工验证：删除播放列表、添加文件、多选移除、排序、刷新保序、重启持久化。
