# Release Notes - 0.1.8-dev

## 重点修复

### 本地 4K 播放稳定性

修复本地 4K HEVC/10bit 视频播放数分钟后画面冻结、声音和进度继续的问题。

本次排查确认部分 Windows 设备在 D3D11/D3D11VA 硬解链路上会出现 GPU device suspended / `0x887a0005` 错误。为保证稳定性，本地媒体播放默认改为：

```text
OpenGL video output + software decoding
```

带来的变化：

1. 本地 4K 播放不再默认使用 D3D11VA 硬解。
2. 窗口和全屏模式使用同一套稳定渲染策略。
3. 全屏切换不再重载播放器或切换渲染器。
4. 画面停滞自动恢复时改用软件解码兼容模式。

### 悬浮时钟恢复

在本地 4K 播放稳定后，恢复全屏播放时的悬浮时钟显示。经验证，悬浮时钟不是冻结根因。

### 诊断日志收敛

排查阶段加入的强制诊断日志已关闭。现在：

1. 普通运行默认不输出大量播放快照。
2. `mpv_runtime.log` 仅在启用诊断日志且级别为“调试”时生成。
3. 保留独立 libmpv 日志能力，便于后续定位复杂播放问题。

## 直播 4K 策略

直播播放策略本次未做全局软件解码修改。

原因是当前已确认的问题发生在本地 4K 文件播放链路，直播 HLS/MPD 使用独立缓存和播放策略。直接对全部直播启用软件解码可能增加 CPU 压力并影响直播体验。

后续如果直播 4K 出现同类 D3D11VA 错误，将增加可选的“直播兼容模式”。

## 验证情况

已完成验证：

1. 本地 4K 窗口模式播放超过 10 分钟未冻结。
2. 本地 4K 全屏模式播放超过 10 分钟未冻结。
3. 恢复悬浮时钟后，全屏播放超过 10 分钟未冻结。

## 注意事项

本地 4K 软件解码更依赖 CPU 性能。低性能设备上可能出现 CPU 占用升高或掉帧，但优先保证不再发生硬解链路导致的画面冻结。
