# 资源收藏与频道收藏需求与实施方案

## 1. 背景与目标

当前播放器已经支持资源目录扫描、频道资源加载、本地视频/音频播放、频道列表分组播放等能力。后续资源目录也需要把 libmpv/Qt 天然支持的图片和 GIF 纳入资源管理范围。随着资源数量增加，用户需要把常用资源文件、本地媒体文件、图片/GIF 资源和单个频道沉淀为收藏，减少反复切换目录、查找频道文件和搜索频道的成本。

本轮目标是在现有“资源库/资源文件列表”面板中增加两个收藏管理入口：

- 资源收藏管理器：收藏频道资源文件、本地视频、本地音频、GIF、图片。
- 频道收藏管理器：收藏单个频道，而不是收藏整个频道文件。

收藏只保存关联关系和必要快照，不删除、不移动、不修改原始资源文件或频道文件内容。

## 2. 需求清单

### 2.1 资源收藏管理器

资源收藏对象包括：

- 频道资源文件：`.cfg`、`.json`、`.m3u`、`.m3u8`、`.txt` 等。
- 本地视频：当前 `utils.media_types` 已支持的视频格式。
- 本地音频：当前 `utils.media_types` 已支持的音频格式。
- GIF：`.gif`，作为独立资源类型显示和筛选。
- 图片：常见静态图片格式，例如 `.jpg`、`.jpeg`、`.png`、`.bmp`、`.webp`、`.tif`、`.tiff` 等。

功能要求：

- 在资源面板中增加 `资源收藏` Tab。
- 可以查看已收藏资源。
- 可以使用收藏资源：频道资源文件执行加载；本地视频/音频/GIF/图片直接交给播放器打开。
- 可以删除收藏关系，但不删除原始文件。
- 收藏数据需要持久化，重启程序后仍可看到。
- 每次进入或刷新资源收藏管理器时检测有效性。
- 如果原文件不存在，标注为 `已失效`，使用时给出友好提示。

### 2.2 频道收藏管理器

频道收藏对象是单个频道，不是频道文件。

功能要求：

- 在资源面板中增加 `频道收藏` Tab。
- 可以查看已收藏频道。
- 可以播放收藏频道。
- 可以删除频道收藏关系，但不删除频道文件中的频道内容。
- 收藏数据需要持久化，重启程序后仍可看到。
- 频道收藏应保存完整频道快照，包含 Manifest、ManifestType、Keys、DrmType、LicenseUrl、Headers、UserAgent、Referer、轨道信息、台标、EPG 标识等。
- 如果来源频道文件不存在，标注 `来源失效`。
- 如果来源文件存在但找不到同一频道，标注 `频道可能已变更`。
- 即使来源失效，也允许尝试播放频道快照。

## 3. 数据持久化方案

新增收藏管理器：

```text
backend/favorites_manager.py
config/favorites.json
```

建议 JSON 结构：

```json
{
  "version": 1,
  "resource_favorites": [],
  "channel_favorites": []
}
```

### 3.1 资源收藏字段

```json
{
  "id": "hash-or-uuid",
  "name": "US.cfg",
  "type": "channel_resource | video | audio | gif | image",
  "path": "F:/.../US.cfg",
  "added_at": "2026-06-21T00:00:00+08:00",
  "last_used_at": "",
  "note": ""
}
```

去重建议：

- 资源收藏按规范化绝对路径去重。
- 路径大小写在 Windows 下按不敏感处理。

### 3.2 频道收藏字段

```json
{
  "id": "hash",
  "name": "广东卫视",
  "category": "国内",
  "source_path": "F:/.../4gtv.txt",
  "source_name": "4gtv",
  "fingerprint": "hash(name + manifest + tvgId)",
  "channel": {
    "Name": "广东卫视",
    "Manifest": "https://...",
    "ManifestType": "hls",
    "Headers": {},
    "UserAgent": "",
    "Referer": ""
  },
  "added_at": "2026-06-21T00:00:00+08:00",
  "last_used_at": ""
}
```

去重建议：

- 优先使用 `TvgId + Manifest`。
- 如果没有 `TvgId`，使用 `Name + Manifest`。
- 对频道快照保存完整字段，避免丢失 DRM、代理、轨道和请求头信息。

## 4. 有效性检测方案

### 4.1 资源收藏有效性

检测规则：

- 文件路径存在：有效。
- 文件路径不存在：已失效。
- 文件存在但类型不在支持列表：类型异常。

显示建议：

- 有效：正常文字。
- 已失效：灰色文字 + `已失效` 标记。
- 类型异常：橙色标记。

### 4.2 频道收藏有效性

检测规则：

- `source_path` 不存在：来源失效。
- `source_path` 存在且能加载：尝试根据 fingerprint 匹配频道。
- 找到匹配频道：有效，可考虑更新快照。
- 未找到匹配频道：频道可能已变更。

播放策略：

- 有效频道：播放最新匹配频道或保存快照。
- 来源失效/频道变更：仍允许尝试播放收藏快照。

## 5. UI 方案

### 5.1 资源面板 Tab 结构

将现有 `NavigationOverlay` 升级为带 Tab 的资源管理面板：

- `资源目录`
- `资源收藏`
- `频道收藏`

现有资源目录功能保持不丢失：

- 打开资源目录。
- 打开在线 m3u。
- 刷新当前目录。
- 全部/频道/视频/音频/GIF/图片筛选。
- 双击资源文件加载或播放。
- 删除选中文件功能仍在资源目录 Tab 中，语义保持为删除原文件。

### 5.2 资源收藏 Tab

建议控件：

- 收藏列表。
- 状态列或状态标签。
- `使用/播放` 按钮。
- `取消收藏` 按钮。
- `刷新状态` 按钮。

双击行为：

- 有效频道资源文件：加载频道资源。
- 有效视频/音频/GIF/图片：直接交给播放器打开。
- 已失效：提示文件不存在。

### 5.3 频道收藏 Tab

建议控件：

- 收藏频道列表。
- 显示频道名、分类、来源、状态。
- `播放` 按钮。
- `取消收藏` 按钮。
- `刷新状态` 按钮。

双击行为：

- 播放该频道收藏快照。

### 5.4 收藏面板检索与分类增强

本轮新增小需求目标：

- 收藏数量增加后，用户可以快速按类型和关键词定位资源或频道。
- 收藏列表展示信息要与 `资源目录` 和频道列表保持一致，减少认知切换。
- 搜索与筛选只影响当前收藏列表显示，不改变收藏数据本身。

#### 5.4.1 资源收藏面板增强

界面新增：

- 在每个资源收藏项前显示资源类型前缀：
  - `[频道]`：频道资源文件。
  - `[视频]`：本地视频文件。
  - `[音频]`：本地音频文件。
  - `[GIF]`：GIF 动图文件。
  - `[图片]`：静态图片文件。
  - `[资源]`：未知或暂未识别类型。
- 增加资源类型筛选单选框，样式与 `资源目录` Tab 保持一致：
  - `全部`
  - `频道`
  - `视频`
  - `音频`
  - `GIF`
  - `图片`
- 增加搜索框：
  - 支持按收藏名称搜索。
  - 支持按文件路径搜索。
  - 建议大小写不敏感。
  - 搜索为空时显示当前筛选条件下的全部收藏。

显示建议：

- 资源收藏项格式建议为：

```text
[频道] CCTV.m3u  [有效]
[视频] demo.mp4  [有效]
[音频] music.flac  [有效]
[GIF] intro.gif  [有效]
[图片] poster.jpg  [有效]
```

- 失效资源仍显示原类型；如果类型无法从路径判断，则显示 `[资源]`。
- 搜索和筛选组合生效，例如先选 `视频`，再搜索 `4K`，只显示视频收藏中匹配 `4K` 的项目。
- GIF 作为独立类型，不归入普通图片筛选；`图片` 筛选只显示静态图片。

#### 5.4.2 频道收藏面板增强

界面新增：

- 增加搜索框：
  - 支持按频道名称搜索。
  - 支持按频道分类搜索。
  - 支持按来源名称搜索。
  - 支持按直播类型搜索，例如 `HLS`、`DASH`、`MPD`。
  - 建议大小写不敏感。
- 在每个频道收藏项前显示直播类型：
  - `[HLS]`：`ManifestType` 为 `hls`，或地址包含 `.m3u8`。
  - `[DASH]`：`ManifestType` 为 `dash`，或地址包含 `.mpd`。
  - `[FLV]`：`ManifestType` 为 `flv`，或地址包含 `.flv`。
  - `[MP4]` / `[TS]`：直链文件型媒体。
  - `[页面]`：页面型源或需要网页嗅探的源。
  - `[直播]`：无法明确判断但仍是频道收藏。

显示建议：

```text
[HLS] 广东卫视 · 国内  [有效]
[DASH] KBSworld · 国际  [有效]
[页面] 某网页频道 · 未分组  [快照可用]
```

直播类型判断建议：

- 优先使用频道快照中的 `ManifestType`。
- 其次根据 `Manifest` URL 后缀或关键字推断。
- 如果频道快照已有 `resolved_info.media_type` 或类似解析结果，可作为补充来源。
- 展示用类型只作为 UI 辅助，不参与播放决策，避免误判影响播放。

#### 5.4.3 技术分析

资源收藏筛选：

- 当前收藏数据应扩展 `type` 字段：`channel_resource`、`video`、`audio`、`gif`、`image`、`unknown`。
- 资源收藏列表应保留一份完整数据 `_resource_favorites`，筛选和搜索只重新渲染列表。
- 筛选条件建议使用 `_resource_favorite_filter`，搜索关键字建议使用 `_resource_favorite_keyword`。
- `utils.media_types` 建议新增：
  - `LOCAL_IMAGE_EXTENSIONS`
  - `LOCAL_GIF_EXTENSIONS`
  - `is_local_image(path)`
  - `is_local_gif(path)`
  - `is_visual_media(path)`，用于后续统一处理图片和 GIF。
- `RESOURCE_EXTENSIONS` 应包含频道、视频、音频、GIF、图片五类资源扩展名。

频道收藏搜索：

- 频道收藏列表应保留完整数据 `_channel_favorites`。
- 搜索关键字建议匹配：
  - `favorite.name`
  - `favorite.category`
  - `favorite.source_name`
  - `favorite.channel.ManifestType`
  - `favorite.channel.Manifest`
  - 推断后的直播类型展示文本

性能建议：

- 收藏列表通常远小于频道列表，初版使用 `QListWidget` 重新渲染即可。
- 搜索框建议使用 `textChanged` 即时过滤。
- 如果后续收藏规模达到数千条，再考虑 `QSortFilterProxyModel` + `QAbstractListModel`。

边界处理：

- 搜索无结果时显示空列表即可，不弹窗。
- 失效项仍参与搜索和筛选。
- 筛选或搜索后，原有 `使用/播放`、`取消收藏`、`刷新状态` 按钮仍作用于当前选中项。
- 取消收藏后，当前筛选和搜索条件不应被重置。

## 6. 添加/取消收藏交互技术评估

### 6.1 方式一：列表行星标收藏

表现：

- 空心五星：未收藏。
- 实心五星：已收藏。
- 点击星标切换收藏状态。

适用位置：

- 资源目录 Tab 的资源文件列表。
- 频道列表面板的频道列表。
- 后续可扩展到收藏 Tab 中用于快速取消收藏。

优点：

- 可发现性最好，用户一眼能看到收藏状态。
- 操作路径最短，适合高频操作。
- 收藏状态直观，不需要右键探索。

技术建议：

- 不建议用 `setItemWidget()` 给每一行塞按钮。频道多时性能、滚动和内存占用都会变差。
- 推荐使用 `QStyledItemDelegate` 自绘星标。
- 在列表点击事件中判断点击坐标是否落在星标区域。
- 星标图标建议使用 Qt Painter 或项目统一 IconButton/SVG 风格，不建议直接依赖 emoji 字体，避免 Windows 字体差异。
- 收藏状态由 `FavoritesManager` 查询，列表刷新时同步空心/实心状态。

风险：

- 需要重构资源列表和频道列表的 item 绘制逻辑。
- 需要处理 hover、selected、disabled/失效状态。
- 需要避免星标区域挤压长文件名或长频道名。

### 6.2 方式二：右键菜单收藏/取消收藏

表现：

- 对资源文件或频道右键。
- 如果未收藏，菜单显示 `添加到收藏`。
- 如果已收藏，菜单显示 `取消收藏`。

优点：

- 实现简单，风险低。
- 不明显改变当前列表布局。
- 适合承载更多操作，例如 `打开所在目录`、`刷新状态`、`查看详情`。

技术建议：

- 对 `QListWidget` 设置 `Qt.CustomContextMenu`。
- 根据右键命中的 item 和当前收藏状态动态生成菜单。
- 资源收藏按规范化路径判断。
- 频道收藏按频道 fingerprint 判断。

风险：

- 可发现性较弱，很多用户不知道可以右键。
- 已收藏状态不直观，仍需要列表有视觉提示。
- 不适合触控场景。

### 6.3 推荐组合方案

推荐采用组合方案：

- 主操作：列表行右侧星标。
- 辅助操作：右键菜单。
- 收藏管理 Tab 中保留按钮：`使用/播放`、`取消收藏`、`刷新状态`。

实施顺序建议：

1. 先实现收藏数据管理和持久化。
2. 先接入右键菜单，快速形成可用闭环。
3. 再实现星标委托，提高可发现性和操作效率。
4. 最后优化星标 hover、tooltip、失效状态样式。

这样可以降低一次性 UI 重构风险，每一步都可验证。

- 添加/删除资源收藏。
- 添加/删除频道收藏。
- 判断资源/频道是否已收藏。

- 增加信号：使用资源收藏、删除资源收藏、播放频道收藏、删除频道收藏、收藏资源等。

修改：

```text
ui/navigation_overlay.py
```

内容：

- 资源收藏 Tab 增加搜索框。

- 频道收藏 Tab 增加搜索框。
- 频道收藏列表项前显示 `[HLS]`、`[DASH]`、`[FLV]`、`[页面]` 等直播类型前缀。
- 新增或复用媒体类型显示辅助函数，避免 UI 中散落判断逻辑。

推荐实现：

- 在 `NavigationOverlay` 中保留完整收藏列表缓存：
  - `_resource_favorites`
  - `_channel_favorites`
- 新增当前过滤状态：
  - `_resource_favorite_filter`
  - `_resource_favorite_keyword`
  - `_channel_favorite_keyword`
- `set_resource_favorites()` 只更新缓存，然后调用 `_apply_resource_favorite_filter()`。
- `set_channel_favorites()` 只更新缓存，然后调用 `_apply_channel_favorite_filter()`。
- 类型前缀建议通过小型 helper 生成，后续可迁移到 `utils.media_types`。
- 资源目录扫描、资源目录筛选、资源收藏筛选要共享同一套扩展名判断，避免同一文件在不同 Tab 中类型不一致。

### 阶段六：验收与优化

验收项：

- 收藏资源文件后，重启程序仍可看到。
- 收藏本地视频/音频/GIF/图片后，重启程序仍可看到并打开。
- 删除原资源文件后，收藏项显示已失效。
- 删除收藏只删除收藏关系，不删除原文件。
- 收藏单个频道后，切换频道文件或重启程序仍可播放收藏频道。
- 删除频道收藏只删除收藏关系，不删除频道文件内容。
- 右键菜单和星标状态保持一致。
- 资源收藏支持类型筛选和搜索。
- 频道收藏支持搜索，并能显示直播类型前缀。

## 9. 风险与注意事项

- 频道收藏必须保存完整频道快照，否则 DRM、Headers、代理、轨道信息容易丢失。
- 星标委托不要使用大量 `setItemWidget()`，否则大列表性能会下降。
- 收藏文件路径需要规范化，避免同一路径因大小写或斜杠差异重复收藏。
- 删除收藏和删除资源文件必须在 UI 文案上严格区分，避免误删原文件。
- 频道来源失效不等于频道不可播放，仍应允许播放快照。
- 图片/GIF 属于资源管理和打开能力增强，实际显示效果以 libmpv/Qt 当前支持能力为准；UI 不应因为无法预览图片而阻断收藏管理。
- 初版不建议做批量收藏，先保证单项收藏稳定。

## 10. 推荐最终方案

最终采用：

- `FavoritesManager + config/favorites.json` 持久化。
- 资源面板增加三个 Tab：`资源目录`、`资源收藏`、`频道收藏`。
- 收藏入口采用“星标 + 右键菜单”的组合。
- 第一版编码优先完成右键菜单闭环，再补星标委托。
- 频道收藏保存完整快照，并提供来源有效性状态。
