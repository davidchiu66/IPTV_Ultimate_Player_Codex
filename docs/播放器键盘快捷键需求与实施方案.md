# 播放器键盘快捷键需求与实施方案

## 需求背景

播放器需要补充两个常用键盘快捷键，降低直播和本地媒体播放时对鼠标控制条的依赖：

1. 空格键：播放/暂停切换。
2. 回车键：窗口/全屏切换。

## 功能范围

### 空格键

- 当前为播放状态时，按空格暂停。
- 当前为暂停状态时，按空格继续播放。
- 本地媒体播放完成后，按空格沿用现有“重新播放”逻辑。
- 直播和本地媒体统一支持。

### 回车键

- 当前为窗口状态时，按回车切换到全屏。
- 当前为全屏状态时，按回车恢复窗口。
- 同时支持主键盘 Return 和小键盘 Enter。

## 需要规避的场景

快捷键不能抢占正常输入和控件操作：

1. 搜索框、输入框、文本框内按空格或回车时，不触发播放器快捷键。
2. 下拉框、数值框、列表、按钮、滑块等控件获得焦点时，不触发播放器快捷键。
3. 模态对话框或弹出菜单打开时，不触发播放器快捷键。
4. 按住键盘产生自动重复时，不连续触发暂停/全屏切换。
5. 现有 Esc、F1、F11 全屏逻辑保持不变。

## 实施方案

1. 在 `ui/main_window.py` 中使用主窗口已安装的 QApplication 事件过滤器统一处理播放器快捷键。
2. 新增 `_handle_player_key_shortcut(event, watched=None)`，集中判断和执行快捷键。
3. 新增焦点过滤函数，判断当前焦点控件是否为输入类或可操作控件。
4. 空格键调用现有 `MpvVideoWidget.toggle_pause()`，复用已有播放/暂停/播完重播逻辑。
5. 回车键调用现有 `_toggle_fullscreen()`，复用现有全屏、光标隐藏、覆盖层重排逻辑。
6. `keyPressEvent()` 也调用同一快捷键处理函数，作为主窗口直接获得焦点时的兜底。

## 验收方法

1. 直播播放时按空格，可以暂停；再次按空格，可以继续。
2. 本地视频播放时按空格，可以暂停；再次按空格，可以继续。
3. 本地视频播放结束后按空格，可以重新开始播放。
4. 窗口状态按回车，进入全屏。
5. 全屏状态按回车，恢复窗口。
6. 搜索框中输入空格，不触发暂停。
7. 输入框或下拉框中按回车，不触发全屏。
8. 资源列表或频道列表获得焦点时按回车，不触发全屏，避免影响列表自身操作。
9. 打开关于、错误提示、确认框等模态对话框时，空格和回车不触发播放器快捷键。
10. 长按空格或回车，不会连续触发暂停/全屏抖动。

## 后续扩展建议

后续可以继续补充：

1. 左右方向键：快退/快进。
2. 上下方向键：音量增减。
3. `M`：静音切换。
4. `F`：全屏切换。
5. 设置页增加快捷键开关和快捷键说明。
