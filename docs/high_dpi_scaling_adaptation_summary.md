# 高 DPI 缩放自适应问题总结与处理备案

记录时间：2026-06-24

## 背景

在 Windows 10/11 高 DPI 场景下，用户反馈 24 寸 2560x1440、150% 缩放时应用启动后窗口接近全屏，标题栏按钮难以操作；进一步验证到 14 寸 1920x1080、175% 缩放时，资源库、设置等自绘面板出现布局压缩、控件重叠、按钮区域变形等问题。

后续修复过程中又暴露两个回归：

- 自绘标题栏在 150%/175% 缩放下可见但按钮点击无效。
- 设置面板新增滚动区域后，滚动内容区露出 Qt 默认浅色背景，破坏玻璃拟态风格。

## 分析结论

1. Qt 逻辑坐标与 Windows 物理坐标混用。
   frameless 窗口的 `WM_NCHITTEST` 使用 Windows 物理屏幕坐标，而原实现用 Qt 的逻辑 `frameGeometry()` 参与判断。在 150%/175% 缩放下会发生命中区域错位，标题栏按钮可能被误判为窗口边缘区域，导致点击事件被系统截走。

2. 主窗口启动尺寸固定。
   原始启动尺寸固定为 `1620x980`，在高缩放小逻辑屏幕上会超过可用区域，看起来像全屏或遮挡标题栏控制按钮。

3. 侧滑面板按大屏固定尺寸设计。
   资源库、设置、频道列表、详情、播放列表等面板原本使用固定宽度、固定边距和固定高度减量。在 175% 缩放下，Qt 可用逻辑高度显著减少，内容没有滚动承载时只能互相挤压。

4. 设置面板内容缺少滚动容器。
   设置项较多，在小逻辑高度下无法完整容纳。新增滚动容器是必要的，但必须显式处理 `QScrollArea`、viewport 和内容 QWidget 的透明背景，否则会露出默认浅色背景。

5. 语言默认值本身未被改坏。
   代码默认语言仍是 `zh_CN`，工作区 `config/app_settings.json` 也为中文。界面显示 English 更可能来自另一个运行目录或安装版配置曾保存过 `en_US`。需要在 UI 层增强兜底，缺失或非法语言值强制回退中文。

## 解决方案

1. 主窗口启动尺寸自适应。
   根据当前屏幕 `availableGeometry()` 计算安全启动尺寸，默认不超过可用区域 90%，避免高缩放下标题栏控制按钮跑出屏幕。

2. 修复自绘标题栏命中测试。
   对不可调整大小的自绘主窗口禁用原生 resize hit-test；可调整大小场景改用 Win32 `GetWindowRect()` 的物理坐标计算边缘命中，避免 DPI 坐标错位。

3. 标题栏按钮事件保护。
   标题栏拖拽和双击最大化逻辑显式避开最小化、最大化、关闭按钮区域，防止父级标题栏吞掉按钮事件。

4. 侧滑面板基础自适应。
   在 `BaseOverlay` 中增加基于父窗口逻辑尺寸的宽度、边距、高度计算；小高度场景自动进入 compact 模式。资源库、设置、频道列表、详情、播放列表等继承面板统一受益。

5. 设置面板滚动化。
   设置表单放入 `QScrollArea`，标题和底部“取消/保存”按钮固定可见。高缩放小屏下压缩边距和行距，避免重叠。

6. 资源库面板紧凑化。
   资源库在 compact 模式下压缩根边距、tab 内边距、按钮 padding、列表项 padding，并保持列表最小可用高度。

7. 设置面板风格修复。
   `QScrollArea`、viewport、内容 QWidget 统一设为透明，保持玻璃拟态深色背景。

8. 语言选择兜底。
   设置面板语言下拉固定显示 `中文 / English`，不依赖当前语言翻译；传入缺失或非法语言值时回退 `zh_CN`。

## 验证记录

已通过以下编译与轻量实例化检查：

```powershell
python -m py_compile ui\base_overlay.py ui\settings_overlay.py ui\navigation_overlay.py ui\channel_list_overlay.py ui\detail_overlay.py ui\playlist_overlay.py
python -m py_compile ui\window_chrome.py ui\main_window.py
```

额外验证：

- `NavigationOverlay` 实例化后 `_button_style()` / `_list_style()` 不再触发 f-string QSS 花括号解析错误。
- `SettingsOverlay` 传入 `zh_CN` 时显示 `中文`。
- `SettingsOverlay` 传入非法语言值时回退 `zh_CN` 并显示 `中文`。

## 后续建议

- 后续新增自绘面板必须继承统一的自适应布局策略，避免再次写死大屏尺寸。
- 对涉及 QSS 的 f-string，必须转义 QSS 花括号或改用 `.format()`/模板函数集中生成。
- 建议补充一个高 DPI UI 冒烟测试清单，覆盖 125%、150%、175% 三档缩放下的主窗口、设置、资源库、频道列表和播放列表。
