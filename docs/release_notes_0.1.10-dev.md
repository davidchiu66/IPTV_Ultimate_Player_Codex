# Release Notes - 0.1.10-dev

## 本次重点

本版本围绕“发布后自更新能力”和“资源/播放列表交互体验”做增强，补齐 GitHub 版本检查、在线更新下载入口，并优化资源库与播放列表面板的易用性。

## 新增功能

### GitHub 版本检查

关于界面的 `检查版本更新` 现在会请求 GitHub latest release，并与本地版本进行比较。

支持：

1. 识别 `v0.1.10`、`0.1.10`、`0.1.10-dev` 等版本格式。
2. 远端版本高于本地版本时提示有新版本。
3. 当前已是最新版本时给出明确提示。
4. 网络失败或 GitHub 不可访问时给出错误提示，并可打开 Release 页面。

### 在线更新

关于界面的 `在线更新` 现在会先检查新版本，再按当前运行形态选择更新包。

支持：

1. 安装版优先选择 Release 中的 Windows 安装包。
2. 便携版优先选择 Release 中的 Portable zip 包。
3. 源码开发模式不会覆盖本地代码，只提示打开 Release 页面手动下载。
4. 下载过程显示进度条、文件名、已下载大小和总大小。
5. 支持取消下载。
6. 下载完成后启动安装器或便携版 updater 前，会提示用户当前播放器将退出。

## 体验优化

### 资源库按钮布局优化

资源库目录页按钮改为更紧凑的两列布局，为资源文件列表释放更多空间。

新的布局：

1. 第一排：`打开资源目录` / `打开在线资源`
2. 第二排：`打开资源文件` / `刷新当前资源`
3. 底部：`删除选中文件` / `取消`

### 打开单个资源文件

资源库新增 `打开资源文件`，可直接选择单个受支持资源：

1. 频道文件：m3u、m3u8、txt、cfg、json
2. 本地视频
3. 本地音频
4. 图片
5. GIF

选择频道文件后加载频道列表；选择本地媒体后直接播放。

### 播放列表专辑下拉提示

播放列表面板的专辑选择框增加右侧下拉区域和箭头提示，避免看起来像普通文本框。多个播放专辑时，用户能更直观看到该控件可展开切换。

## 修复问题

### 版本检查跨线程 UI 崩溃

修复点击 `检查版本更新` 后因后台线程直接更新状态栏和弹窗导致的 Qt 跨线程错误：

```text
QObject: Cannot create children for a parent that is in a different thread
```

版本检查和下载结果现在通过 Qt 信号回到主线程处理 UI。

### About 界面交互调整

点击 `检查版本更新` 或 `在线更新` 时，About 界面不再自动关闭，结果弹窗会叠加显示，用户体验更自然。

## 技术调整

1. 新增 `utils/update_manager.py`，集中处理版本比较、Release 查询、资产选择和下载。
2. 新增在线更新下载进度弹窗，统一玻璃拟态风格。
3. 安装版更新通过启动安装器完成。
4. 便携版更新通过外部 PowerShell updater 脚本在程序退出后解压覆盖。
5. 资源库刷新语义从“当前目录”扩展为“当前资源”，兼容单文件上下文。

## 验收建议

1. 点击 About 图标，打开关于界面。
2. 点击 `检查版本更新`，About 不关闭，并弹出检查结果。
3. 断网或 GitHub 不可访问时，检查版本更新应提示失败，不应卡死。
4. 点击 `在线更新`，有新版本时可进入下载确认。
5. 下载过程中应显示进度，并可取消。
6. 资源库面板按钮应按两列布局显示。
7. `打开资源文件` 可打开单个频道文件或本地媒体文件。
8. 播放列表专辑选择框右侧应出现明显下拉提示。
