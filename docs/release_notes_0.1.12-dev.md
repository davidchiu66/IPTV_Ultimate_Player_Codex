# Release Notes - 0.1.12-dev

## 版本重点

本版本重点收敛浏览器播放链路：浏览器播放现在优先复用与 libmpv 一致的真实地址解析能力，先获取真实媒体地址和请求上下文，再交给 HLS / DASH / FLV / 原生播放器播放；只有直接播放失败或无法解析真实媒体时，才回退到网页 iframe 容器。

同时针对 GDTV、IPTV345 等需要页面探测或特殊请求上下文的频道做了兼容修复，并优化播放态控制面板的边界触发体验。

## 播放策略调整

### 浏览器播放优先真实媒体地址

浏览器播放器不再直接把频道页面当作 iframe 打开，而是先调用统一解析接口：

1. `/api/resolve` 新增 `probe=1` 和播放会话参数，支持必要时触发后端 WebEngine 探测。
2. 解析结果会带回真实媒体地址、媒体类型、Referer、User-Agent、Headers、原始页面来源等上下文。
3. 前端根据解析后的 `ManifestType` 选择 HLS、DASH、FLV 或原生播放。
4. iframe 仅作为最后兜底，在 HLS 等播放链路失败后启用。

### 前端播放失败兜底

浏览器播放器增强了错误处理：

1. HLS 清单加载超时、代理返回 403/5xx、网络错误或媒体引擎致命错误时，会优先尝试刷新解析。
2. 已经解析到页面来源但媒体播放失败时，会自动切换到 iframe 兜底。
3. 新增解析请求超时和播放会话取消逻辑，避免旧频道的长耗时解析结果覆盖当前播放。
4. 本地代理 URL 构造统一携带必要上下文，并根据站点策略过滤不适合透传的敏感头。

## 站点兼容修复

### GDTV 频道

针对 GDTV 频道 libmpv 可播放、浏览器播放 403 或无法进入 iframe 兜底的问题，本版本做了专门修复：

1. 后端新增 GDTV API 链路解析，尝试从频道详情接口和节点参数接口中解析真实直播地址。
2. WebEngine 探测时补充 GDTV 相关的浏览器请求头、Accept-Language、Client Hints 和 Cookie 捕获。
3. GDTV/tcdn 直播媒体请求不再透传会导致 CDN 拒绝的 Origin/Cookie 等敏感头。
4. 本地代理避免对已解码的签名 URL 再次 `unquote`，防止 `%2F`、`%2B`、`%3D` 等签名字符被破坏导致 403。
5. 代理侧 403 等错误现在会触发前端播放失败处理和 iframe 兜底，而不是直接停在错误提示。

### IPTV345 频道

针对 IPTV345 央视等分组在 libmpv 正常、浏览器一直解析的问题，本版本修复静态页面误判：

1. 静态解析会过滤 `.js`、`.css`、`.svg`、图片、字体、`.swf` 等明显静态资源。
2. 避免把 `pvjs.js`、`arrow.svg` 之类页面资产误认为媒体候选。
3. 当静态页面解析返回的是页面派生但非真实媒体地址时，浏览器播放会继续进入 WebEngine 探测。
4. 浏览器最终应能拿到与 libmpv 类似的 `index.m3u8?session=...` 真实 HLS 地址。

## libmpv 与代理优化

### libmpv 请求上下文

1. mpv HTTP header 字段增加单行清洗和 mpv 列表转义，避免逗号、换行等字符破坏参数。
2. 自动避免重复设置 User-Agent、Referer 等头。
3. GDTV 类播放链路补充更接近浏览器的 Referer、Origin、Accept-Language 和 Client Hints。
4. 支持在特定失败场景下关闭播放代理并直连重试。
5. 播放失败日志会附带 mpv 运行日志中的关键失败提示，便于定位 HTTP 403、打开失败等问题。

### 本地代理

1. `/api/resolve` 支持深度探测、强制刷新和播放会话取消。
2. `/proxy` 只透传安全的请求头集合，减少浏览器请求头污染上游媒体源。
3. 支持按媒体源携带 User-Agent、Referer、Origin、Cookie 等上下文。
4. 对 GDTV/tcdn 媒体链路自动跳过容易触发 403 的敏感头。

## 界面与交互优化

### 播放态控制面板边界触发

“面板滑入操作模式 = 鼠标移动触发”时，播放中不再因为鼠标在画面中央轻微移动就唤起顶部条和底部控制条。

新的触发规则：

1. 鼠标进入播放区域顶部边界附近时，显示顶部条和底部控制条。
2. 鼠标进入底部控制条触发区时，显示顶部条和底部控制条。
3. 控制条已经显示时，鼠标停留在控制条本体上会继续保持显示。
4. 点击触发模式、侧边资源面板、频道列表和播放列表逻辑不受影响。

### 在线资源加载

在线资源请求遇到 JavaScript Cookie 挑战页时，可通过 QWebEngine 加载并等待重定向 / Cookie 生效后再读取内容，提升部分在线源列表的可用性。

## 技术调整

1. `backend/browser_probe.py` 增强媒体候选捕获、Cookie 捕获、请求头覆盖、取消探测和站点特化处理。
2. `backend/stream_resolver.py` 增加静态资产过滤、JSON 媒体候选提取、GDTV API 解析策略和取消回调。
3. `backend/proxy_server.py` 增强 `/api/resolve` 参数和 `/proxy` 请求头控制。
4. `ui/main_window.py` 增加浏览器播放解析会话、GUI 线程 WebEngine 探测桥接、探测结果回填和失败兜底路由。
5. `frontend/template.html` 增加解析超时、HLS 超时、刷新解析、iframe 兜底和代理上下文构造逻辑。
6. `ui/player_panel.py` 增强 libmpv HTTP 参数处理、失败诊断、GDTV 播放上下文和控制条边界触发。

## 验收建议

1. 选择 GDTV 频道，分别使用 libmpv 和浏览器播放，确认浏览器先解析真实 HLS 地址，HLS 失败时能进入 iframe 兜底。
2. 选择 IPTV345 央视分组频道，确认浏览器不再停留在“解析中”，并能获取真实 `m3u8` 地址播放。
3. 选择韩国 KBS 等已正常频道，确认浏览器播放链路没有回退。
4. 对需要本地代理的 HLS 源测试 403、超时、刷新解析和 iframe 兜底行为。
5. 播放态下在画面中央移动鼠标，确认控制条不再频繁弹出；移动到顶部或底部边界时确认控制条正常显示。
6. 运行安装包和免安装包发布工作流，确认产物版本为 `0.1.12-dev` 且 Release Notes 路径正确。
