# 面板点击滑入操作模式需求与实施方案

## 一、需求背景

当前播放器的面板交互主要依赖鼠标移动到边缘后自动滑入：

1. 鼠标移动到左边缘：根据播放上下文滑入频道列表或资源库。
2. 鼠标移动到右边缘：滑入播放列表。
3. 鼠标移动到顶部边缘：滑入工具栏。
4. 播放状态下移动鼠标：自动显示播放器顶部条和底部控制面板，并在超时后自动隐藏。

这种模式适合熟练用户，但也存在几个问题：

1. 面板可能被误触发，特别是播放状态下鼠标经过边缘时。
2. 面板自动滑入/滑出不够可控。
3. 对新用户来说，边缘触发区域不可见，发现成本较高。
4. 用户希望能选择“显式点击图标”来控制面板开关。

因此新增“操作模式配置项”，允许用户在系统设置中选择：

1. 鼠标移动面板滑入模式：保持当前行为。
2. 鼠标点击面板滑入模式：新增行为，通过可见的小图标点击控制面板滑入和滑出。

## 二、需求理解

### 2.1 鼠标移动面板滑入模式

这是当前默认模式，保持现有逻辑不变：

1. 边缘 HoverZone 继续生效。
2. 工具栏、资源库/频道列表、播放列表仍由鼠标移动触发。
3. 播放器顶部条和底部控制栏仍由鼠标移动自动显示，并按现有超时规则隐藏。
4. 现有面板互斥、层级、播放控制栏压制逻辑继续保留。

### 2.2 鼠标点击面板滑入模式

新增显式点击模式。该模式下：

1. 不再通过鼠标移动触发各面板自动滑入。
2. 不再通过鼠标移动自动显示/隐藏播放器顶部条和底部控制栏。
3. 使用固定在边缘中间位置的小图标作为触发器。
4. 点击图标后，对应面板滑入；再次点击对应图标，面板滑出。
5. 图标方向根据面板状态实时切换。

## 三、交互规则

### 3.1 播放器未开启状态

播放器未开启指当前未处于实际播放状态，或播放器处于待机/占位页状态。

界面边缘显示三个点击触发图标：

| 位置 | 初始图标 | 点击后动作 | 面板滑入后图标 | 再次点击 |
|---|---|---|---|---|
| 左侧中间 | `>` | 资源库面板滑入 | `<` | 资源库面板滑出 |
| 右侧中间 | `<` | 播放列表面板滑入 | `>` | 播放列表面板滑出 |
| 顶部中间 | `﹀` | 工具栏滑入 | `︿` | 工具栏滑出 |

未播放状态下不显示底部播放器控制面板触发图标。

### 3.2 播放器开启状态

播放器开启指已经进入播放态，包括直播、本地视频、本地音频、图片/GIF 展示等播放器工作态。

界面边缘显示四个点击触发图标：

| 位置 | 初始图标 | 点击后动作 | 面板滑入后图标 | 再次点击 |
|---|---|---|---|---|
| 左侧中间 | `>` | 资源库面板滑入 | `<` | 资源库面板滑出 |
| 右侧中间 | `<` | 播放列表面板滑入 | `>` | 播放列表面板滑出 |
| 顶部中间 | `﹀` | 播放器顶部条滑入 | `︿` | 播放器顶部条滑出 |
| 底部中间 | `︿` | 播放器控制面板滑入 | `﹀` | 播放器控制面板滑出 |

注意：播放状态下，顶部中间图标控制的是“播放器顶部条”，不是全局工具栏。全局工具栏在播放状态下原则上不通过顶部点击图标直接打开，以避免和播放器顶部条职责混淆。

### 3.3 左侧图标与频道列表的关系

用户本次需求明确要求左侧点击图标控制“资源库面板滑入”。因此点击模式下左侧图标统一控制资源库，不复用当前 Hover 左边缘的“直播时打开频道列表、本地播放时打开资源库”规则。

频道列表仍可通过播放器顶部条、资源库内频道资源、频道收藏或其他已有入口进入。

如果后续希望点击模式下左侧图标也支持“直播时频道列表、本地时资源库”，可以作为二阶段增强，但本次方案按用户原始描述实现为资源库。

## 四、系统设置设计

### 4.1 配置项名称

系统设置新增：

`面板滑入操作模式`

选项：

1. `鼠标移动触发`
2. `鼠标点击触发`

默认值：

`鼠标移动触发`

### 4.2 持久化位置

建议新增 `utils/panel_interaction_settings.py`，复用现有 `utils.proxy_settings.load_settings/save_settings`。

配置结构建议：

```json
{
  "ui": {
    "panel_interaction_mode": "hover"
  }
}
```

取值：

1. `hover`：鼠标移动面板滑入模式。
2. `click`：鼠标点击面板滑入模式。

提供接口：

```python
DEFAULT_PANEL_INTERACTION_MODE = "hover"

def get_panel_interaction_mode() -> str:
    ...

def set_panel_interaction_mode(mode: str) -> None:
    ...
```

## 五、UI 组件设计

### 5.1 新增点击触发器组件

建议新增文件：

`ui/edge_trigger_buttons.py`

核心类：

```python
class EdgeTriggerButton(QPushButton):
    clicked = Signal(str)
```

或使用 `QWidget + paintEvent` 自绘，以便保持播放器控制面板的玻璃拟态风格。

建议采用自绘 QWidget，原因：

1. 可以避免系统按钮样式。
2. 更容易做半透明背景、蓝色描边和轻微辉光。
3. 与当前 Apple TV 风格控制栏更一致。

组件属性：

1. `edge`: `left/right/top/bottom`
2. `expanded`: 当前对应面板是否打开
3. `active`: 是否可见/可点击

视觉建议：

1. 尺寸：约 `34x58`，顶部/底部约 `58x34`。
2. 背景：深色半透明玻璃。
3. 边框：柔和蓝色描边。
4. hover：增强蓝色辉光。
5. 图标：使用文本符号 `>`、`<`、`﹀`、`︿` 或后续改为 Material Symbols。
6. 层级：高于悬浮时钟、播放区域、加载蒙层，低于已滑入的面板主体。

### 5.2 触发器容器

建议在 `MainWindow` 中创建四个触发器，父级挂载到：

`self.player_panel.video_stack_host`

原因：

1. 当前所有主要 overlay 都挂在该 host 上。
2. 能保持与播放器区域同层级坐标。
3. 窗口大小变化、全屏切换时便于统一 reposition。

方法建议：

```python
def _create_edge_click_triggers(self): ...
def _position_edge_click_triggers(self): ...
def _update_edge_click_triggers(self): ...
def _set_panel_interaction_mode(self, mode): ...
```

窗口 resize、全屏切换、播放状态变化、面板显示/隐藏后，都调用 `_update_edge_click_triggers()`。

## 六、行为实现方案

### 6.1 模式切换

当配置为 `hover`：

1. 显示行为保持当前逻辑。
2. 隐藏点击触发器。
3. 启用 `PlayerPanel` 的 `HoverZone`。
4. 允许播放器控制条由鼠标移动自动显示。
5. 保留工具栏自动超时隐藏和播放控制栏自动超时隐藏。

当配置为 `click`：

1. 显示点击触发器。
2. 禁用 `PlayerPanel` 的 `HoverZone` 鼠标触发。
3. 禁止 `PlayerPanel.mouseMoveEvent/_poll_cursor` 自动显示控制栏。
4. 侧边面板和工具栏不再因为鼠标移动自动滑入。
5. 面板是否显示完全由触发图标点击决定。
6. 如果用户点击其他入口，如工具栏按钮、播放器菜单按钮、设置按钮，仍应保持原有功能。

### 6.2 面板开关方法

建议新增显式 toggle 方法，避免在点击模式下复用只负责 show 的方法：

```python
def _toggle_navigation_overlay(self): ...
def _toggle_playlist_overlay(self): ...
def _toggle_toolbar_overlay(self): ...
def _toggle_player_top_bar(self): ...
def _toggle_player_bottom_bar(self): ...
```

对于侧边面板：

1. 如果对应面板可见：调用 `hide_with_animation()`。
2. 如果不可见：调用现有 `_show_navigation()` 或 `_show_playlist_overlay()`。
3. 面板显示/隐藏后更新对应图标方向。

对于播放器顶部条/底部控制栏：

建议在 `PlayerPanel` 内新增公开方法：

```python
def toggle_top_bar(self): ...
def toggle_bottom_bar(self): ...
def show_top_bar_manual(self): ...
def hide_top_bar_manual(self): ...
def show_bottom_bar_manual(self): ...
def hide_bottom_bar_manual(self): ...
def set_panel_interaction_mode(self, mode): ...
```

点击模式下：

1. 不启动 `_controls_hide_timer`。
2. 用户点击底部图标显示底部控制栏后，控制栏保持显示，直到用户再次点击隐藏。
3. 用户点击顶部图标显示播放器顶部条后，顶部条保持显示，直到用户再次点击隐藏。

### 6.3 与现有面板互斥规则

现有侧边面板互斥关系建议保持：

1. 资源库显示时，隐藏频道列表、详情、设置、播放列表。
2. 播放列表显示时，隐藏资源库、频道列表、详情、设置。
3. 设置显示时，隐藏资源库、频道列表、详情、播放列表。

点击模式不改变这些互斥关系，只改变“触发方式”。

对于播放器顶部条和底部控制栏：

1. 它们可以与资源库/播放列表同时存在，但侧边面板显示时，仍建议压制底部控制栏，避免遮挡。
2. 如果侧边面板打开，底部控制栏可以自动隐藏，但不要改变触发器状态；侧边面板关闭后，触发器根据实际可见状态刷新。

### 6.4 播放状态判断

建议复用已有状态：

1. `self.player_panel._playback_active`
2. `self.player_panel.mpv_widget is not None`
3. `self.current_channel`

封装为：

```python
def _is_player_active_for_triggers(self) -> bool:
    ...
```

判断标准建议：

1. 正在播放或曾经进入播放界面，显示四个触发器。
2. 停止播放并回到占位页后，只显示三个触发器。
3. 加载中视为播放开启状态，因为用户此时需要控制播放相关面板。

## 七、影响范围

### 7.1 需要修改的模块

1. `ui/settings_overlay.py`
   - 新增操作模式下拉框。
   - 扩展 `settings_saved` 信号参数。
   - `set_values()` 支持传入当前模式。

2. `utils/panel_interaction_settings.py`
   - 新增配置读写工具。

3. `ui/main_window.py`
   - 加载/保存操作模式。
   - 创建点击触发器。
   - 切换 hover/click 模式。
   - 实现各类 toggle 方法。
   - resize/fullscreen/overlay 状态变化时更新触发器位置和方向。

4. `ui/player_panel.py`
   - 支持禁用鼠标移动自动显示控制栏。
   - 暴露手动显示/隐藏播放器顶部条和底部控制栏的方法。
   - 点击模式下不启动自动隐藏计时器。

5. `ui/edge_trigger_buttons.py`

### 7.2 不应修改的范围

2. 不改变频道/资源/播放列表数据逻辑。
3. 不改变已有信号槽的业务含义。
4. 不删除当前鼠标移动模式。

## 八、风险与处理

### 8.1 点击触发器被 overlay 遮挡

风险：资源库/播放列表/工具栏滑入后可能遮挡触发器，导致无法点击关闭。

方案：

1. 触发器层级应高于普通播放区域。
2. 但对于已滑入面板，触发器可以放在面板边缘外侧可见区域，或者在面板显示后 raise。
3. 每次 overlay show/hide 后调用 `_raise_edge_click_triggers()`。

### 8.2 与悬浮时钟层级冲突

风险：右侧播放列表曾经被悬浮时钟遮挡，新增触发器也可能冲突。

方案：

1. 点击触发器层级高于悬浮时钟。
2. 面板主体层级高于触发器。
3. 播放列表显示时继续保持高于悬浮时钟。

### 8.3 点击模式下自动隐藏逻辑残留

风险：用户点击显示控制栏后，现有 `_controls_hide_timer` 又把它隐藏。

方案：

1. `PlayerPanel` 增加 `_panel_interaction_mode`。
2. `_show_controls()` 在 click 模式下不启动自动隐藏计时器。
3. `_poll_cursor()` 在 click 模式下直接 return。
4. `mouseMoveEvent()` 在 click 模式下不触发 `_show_controls()`。

### 8.4 播放状态切换后图标数量不更新

风险：停止播放后底部图标仍存在，或开始播放后底部图标未出现。

方案：

1. 播放开始、停止、加载中、播放失败、播放完成后统一调用 `_update_edge_click_triggers()`。
2. `PlayerPanel` 可新增状态变化信号，或在 `MainWindow` 现有播放流程节点手动调用。

## 九、实施步骤

### 阶段一：配置层

1. 新增 `utils/panel_interaction_settings.py`。
2. 设置面板新增“面板滑入操作模式”下拉框。
3. `MainWindow._show_settings()` 传入当前模式。
4. `MainWindow._on_settings_saved()` 保存模式并立即应用。

### 阶段二：触发器 UI

1. 新增 `EdgeTriggerButton`。
2. `MainWindow` 创建左、右、上、下四个触发器。
3. 实现 resize/fullscreen 下的位置计算。
4. 实现 hover/click 模式下显示隐藏触发器。

### 阶段三：面板 toggle

1. 实现资源库、播放列表、工具栏 toggle。
2. `PlayerPanel` 暴露顶部条、底部控制栏手动 toggle。
3. 点击触发器绑定对应 toggle。
4. 根据面板可见状态更新图标方向。

### 阶段四：禁用自动滑入

1. click 模式禁用 HoverZone。
2. click 模式禁用播放器控制栏鼠标移动自动显示。
3. click 模式禁用控制栏自动隐藏。
4. hover 模式恢复现有行为。

### 阶段五：联调与收尾

1. 检查播放态/非播放态图标数量。
2. 检查面板互斥和层级。
3. 检查设置保存后立即生效。
4. 检查重启后配置持久化。

## 十、验收方法

### 10.1 设置项验收

1. 打开系统设置，应看到“面板滑入操作模式”配置项。
2. 默认值应为“鼠标移动触发”。
3. 切换到“鼠标点击触发”并保存，应立即生效。
4. 重启程序后，设置应保持。

### 10.2 鼠标移动模式验收

1. 选择“鼠标移动触发”。
2. 鼠标移动到左边缘，仍按现有规则滑入频道列表或资源库。
3. 鼠标移动到右边缘，播放列表滑入。
4. 鼠标移动到顶部，工具栏滑入。
5. 播放状态下移动鼠标，播放器顶部条和底部控制栏自动显示并超时隐藏。
6. 边缘点击小图标不显示。

### 10.3 鼠标点击模式：未播放状态

1. 选择“鼠标点击触发”。
2. 停止播放，回到待机/占位页。
3. 左侧中间显示 `>`，点击后资源库滑入，图标变 `<`。
4. 再次点击左侧图标，资源库滑出，图标恢复 `>`。
5. 右侧中间显示 `<`，点击后播放列表滑入，图标变 `>`。
6. 再次点击右侧图标，播放列表滑出，图标恢复 `<`。
7. 顶部中间显示 `﹀`，点击后工具栏滑入，图标变 `︿`。
8. 再次点击顶部图标，工具栏滑出，图标恢复 `﹀`。
9. 底部不应显示播放器控制栏触发图标。

### 10.4 鼠标点击模式：播放状态

1. 开始播放直播或本地资源。
2. 左侧、右侧、顶部、底部四个图标均显示。
3. 左侧图标控制资源库滑入/滑出。
4. 右侧图标控制播放列表滑入/滑出。
5. 顶部图标控制播放器顶部条显示/隐藏。
6. 底部图标控制播放器控制面板显示/隐藏。
7. 鼠标移动到边缘不应自动滑入任何面板。
8. 鼠标移动不应自动显示播放器控制栏。
9. 手动显示的播放器顶部条/底部控制栏不应自动超时隐藏。

### 10.5 层级验收

1. 点击触发器不应被悬浮时钟遮挡。
2. 点击触发器不应被加载蒙层遮挡。
3. 资源库/播放列表滑入后仍可通过对应图标滑出。
4. 全屏和窗口模式下触发器位置都正确。
5. 高 DPI 缩放下触发器不变形、不越界。

## 十一、结论

该需求可在现有架构上实现，不需要重构播放器核心逻辑。推荐采用“配置持久化 + EdgeTriggerButton 自绘控件 + MainWindow 统一调度 + PlayerPanel 支持手动控制”的方案。

本次编码应严格控制在 UI 交互层：

1. 保留当前鼠标移动模式。
2. 新增点击触发模式。
3. 不改变播放、解析、收藏、播放列表等业务逻辑。
4. 不删除现有信号槽，只新增必要的模式控制和手动 toggle 接口。
