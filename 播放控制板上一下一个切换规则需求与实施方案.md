# 播放控制板上一下一个切换规则需求与实施方案

## 1. 背景

当前播放控制板上的“上一个”和“下一个”按钮已经有信号链路，但实际语义偏向“当前频道列表中的上/下一个频道”。当播放来源不是普通频道列表时，例如从频道收藏播放直播，或者播放本地视频/音频/图片/GIF 资源，按钮缺少对应的播放队列上下文，因此用户感知为按钮没有发挥作用。

本次改动目标是让控制板上的“上一个/下一个”根据当前播放来源自动选择正确队列：

- 普通频道列表直播：在当前频道文件加载出的频道列表中切换。
- 频道收藏直播：在频道收藏列表中切换。
- 本地资源播放：在当前资源目录列表中切换本地资源文件。

## 2. 需求理解

### 2.1 直播从频道收藏进入

当用户在“频道收藏”Tab 中播放某个收藏频道时，播放器应记录当前播放上下文为 `channel_favorites`。

此时点击：

- `上一个`：播放频道收藏列表中的上一个可播放频道。
- `下一个`：播放频道收藏列表中的下一个可播放频道。

切换范围应以当前频道收藏管理器中稳定的收藏列表为准。若列表经过搜索筛选，建议第一阶段仍使用完整频道收藏列表，避免播放队列随搜索框变化而产生不确定行为；后续可扩展为“按当前筛选结果切换”。

频道收藏中标注为来源失效或频道可能已变更的项目，仍允许尝试播放其频道快照；如果播放失败，沿用现有失败提示和重试/播放下一个逻辑。

### 2.2 直播从频道列表进入

当用户在普通频道列表中播放频道，或加载频道资源后从频道分组/频道列表播放，播放器应记录当前播放上下文为 `channel_list`。

此时点击：

- `上一个`：在 `playlist_mgr.streams` 中找当前频道的上一个频道。
- `下一个`：在 `playlist_mgr.streams` 中找当前频道的下一个频道。

保留现有回绕逻辑：到列表末尾后切回第一项，到列表开头前切到最后一项。

### 2.3 本地资源播放

当用户从资源目录、资源收藏中的本地视频/音频/GIF/图片播放时，播放器应记录当前播放上下文为 `resource_files`。

此时点击：

- `上一个`：在当前资源目录扫描结果中找上一个资源文件。
- `下一个`：在当前资源目录扫描结果中找下一个资源文件。

资源范围建议使用当前资源面板中的完整资源文件列表，即当前目录扫描得到的 `cfg_files_cache` 或等价缓存，且只包含播放器支持的资源类型：

- 频道资源文件
- 视频
- 音频
- GIF
- 图片

若当前播放的是本地媒体文件，切到另一个本地媒体文件时直接播放；若切到频道资源文件，则加载该频道资源并滑出频道列表，保持当前资源面板的既有行为。

若当前本地资源来自“资源收藏”，但该资源不在当前资源目录缓存中，第一阶段建议优先在“资源收藏列表”内切换；如果无法建立收藏队列，则退回当前资源目录列表。

## 3. 设计原则

- 不改变控制板 UI，不新增按钮，只补齐“上一个/下一个”的上下文语义。
- 不删除现有频道列表切换逻辑，只把它纳入统一的播放队列调度。
- 播放队列上下文由播放入口设置，而不是由按钮点击时猜测。
- 队列查找尽量使用稳定 ID 或指纹，避免同名频道/同名文件误判。
- 切换失败时给出状态栏提示，不让按钮静默无反应。
- 本次只实现线性上/下一个；随机播放、循环模式、按筛选结果播放等留作后续增强。

## 4. 实施方案

### 4.1 增加播放来源上下文

在 `MainWindow` 增加播放队列上下文字段，例如：

```python
self._playback_queue_context = {
    "kind": "",
    "items": [],
    "current_key": "",
    "source": "",
}
```

建议的 `kind`：

- `channel_list`
- `channel_favorites`
- `resource_files`
- `resource_favorites`

### 4.2 统一生成队列 key

频道 key：

- 优先使用 `FavoritesManager.channel_fingerprint(channel)`。
- 对普通频道列表也复用同一指纹算法。

资源 key：

- 使用规范化绝对路径：`os.path.normcase(os.path.normpath(os.path.abspath(path)))`。

### 4.3 在播放入口设置上下文

需要覆盖的入口：

- 普通频道播放：`MainWindow.play_channel(channel)` 或频道列表选择回调。
- 频道收藏播放：`MainWindow._on_channel_favorite_selected(channel, favorite_id)`。
- 资源目录文件播放：`MainWindow._on_file_selected(path)`。
- 本地资源播放：`MainWindow._play_local_media_file(path)`。
- 资源收藏播放：`MainWindow._on_resource_favorite_selected(path, favorite_id)`。

播放入口设置上下文后再调用现有播放或加载逻辑。

### 4.4 改造上/下一个调度

保留 `player_panel.prev_channel_requested` 和 `player_panel.next_channel_requested` 信号连接，但把 `MainWindow._play_prev_channel()` / `_play_next_channel()` 改为通用调度：

```python
def _play_previous_item(self):
    self._play_adjacent_item(-1)

def _play_next_item(self):
    self._play_adjacent_item(1)
```

`_play_adjacent_item(direction)` 根据 `self._playback_queue_context["kind"]` 分发：

- `channel_list`：从 `playlist_mgr.streams` 切换。
- `channel_favorites`：从收藏管理器返回的频道收藏切换。
- `resource_files`：从当前资源目录缓存切换。
- `resource_favorites`：从资源收藏列表切换。

若当前上下文为空或找不到当前项：

- 直播且 `playlist_mgr.streams` 非空：退回频道列表切换。
- 本地资源且资源目录缓存非空：退回资源目录切换。
- 否则状态栏提示“没有可切换的上/下一个项目”。

### 4.5 与现有失败处理协同

现有失败弹窗中“播放下一个频道”仍可调用 `_play_next_channel()`，但实际会按当前上下文找下一个：

- 普通频道列表失败：切频道列表下一个。
- 频道收藏失败：切频道收藏下一个。
- 本地资源失败：切资源队列下一个。

如果希望失败弹窗文字更准确，后续可把“播放下一个频道”改为“播放下一个”，但本次可以先保持文案不动，降低改动面。

## 5. 边界情况

- 队列只有 1 项：点击上/下一个后仍回到自身没有意义，建议提示“当前队列没有其他项目”。
- 当前项已被取消收藏：重建收藏队列后找不到当前项，提示并退回可用队列。
- 本地文件被删除：跳过不存在文件并提示，或沿用现有“资源失效”提示。
- 频道收藏来源失效：不阻止切换，仍尝试播放快照。
- 正在加载直播时点击上/下一个：沿用当前异步解析取消机制，取消旧请求并启动新请求。
- 当前资源面板有筛选条件：第一阶段队列使用完整列表；后续再考虑按筛选结果切换。

## 6. 验收办法

### 6.1 普通频道列表

1. 打开一个频道资源文件。
2. 从频道列表中播放中间某个频道。
3. 点击播放控制板 `下一个`。
4. 应播放频道列表中的下一个频道，顶部频道名同步刷新。
5. 点击 `上一个`。
6. 应回到前一个频道。
7. 在第一项点击 `上一个`、最后一项点击 `下一个`，应按现有逻辑回绕。

### 6.2 频道收藏

1. 在频道收藏 Tab 中准备至少 3 个收藏频道。
2. 从第二个收藏频道开始播放。
3. 点击 `下一个`，应播放频道收藏列表中的第三个收藏频道。
4. 点击 `上一个`，应回到第二个收藏频道。
5. 不应跳到当前频道文件的频道列表。

### 6.3 本地资源目录

1. 打开一个包含多个视频/音频/GIF/图片的资源目录。
2. 双击播放其中一个本地视频。
3. 点击 `下一个`，应播放当前资源目录中的下一个支持资源。
4. 点击 `上一个`，应播放上一个支持资源。
5. 如果下一个是频道资源文件，应加载频道资源并显示频道列表，而不是崩溃或无响应。

### 6.4 资源收藏

1. 在资源收藏 Tab 中准备至少 3 个本地资源收藏。
2. 从资源收藏中播放第二个资源。
3. 点击 `下一个`，优先播放资源收藏列表中的第三个资源。
4. 点击 `上一个`，回到第二个资源。
5. 若收藏文件已失效，应给出友好提示，并允许继续切换其他资源。

### 6.5 加载中切换

1. 播放一个需要较长解析时间的直播频道。
2. 加载过程中点击 `下一个`。
3. 旧解析任务应被取消或标记过期，新频道开始加载。
4. UI 不应卡死，资源面板/频道面板仍可滑入。

## 7. 后续增强

- 增加播放模式：顺序、单项循环、列表循环、随机。
- 队列可视化：在顶部条或资源面板中显示当前队列来源。
- 上/下一个按当前筛选结果切换。
- 本地资源自动连播。
- 对失效收藏提供“一键清理失效项”。
